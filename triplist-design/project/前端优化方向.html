把"4 主题"真正落地

README 写的是承诺，代码写的是事实 — 让事实赶上承诺。

当前只有一套 tegami（手紙）主题在跑，其它三套（杂志 / 明信片 / 极简）只在文档里出现。建议把它做成一个真正的 design token 切换系统，每套主题不只是换色，而是换整套视觉气质：

  • 手紙 — 当前样式（暖橘 + 衬线 + 圆润）。温暖、私人。
  • 杂志 — 黑白高对比 + 大号无衬线 + 紧密栅格 + 红色 accent。冷峻、编辑感。
  • 明信片 — 米黄牛皮纸底 + 邮戳印章 + 旋转票根 + 衬线题图。复古、旅行感。
  • 极简 — 纯白 + 极淡灰 + 单色 accent + 大量留白。专注、当代。
  • 切换入口放在"我的"或长按 logo，切换有翻页/翻牌过渡动画，不是闪一下完事。
主题不该是"换皮肤"，应该像换刊物 — 同样的内容，不同的"编辑"。 — Design intent

专属插画式图标体系

告别 emoji 大杂烩 — 一套有"手感"的双色线条图标。

当前 emoji 散落在很多地方：📋 攻略、💰 开销、🎒 清单、✨ AI、📸 等等。这造成两个问题：跨平台渲染不一致（苹果/安卓/微信小程序差异巨大）、风格和 app 的"手紙杂志感"完全不搭。

建议做一套统一的线条 + 暖色填充双色图标，~24 个就够覆盖：导航 tab × 4、行程内容类型 × 4、行李分类 × 8、操作 × 6、状态 × 4。每个图标都用同一支"笔"画出来。

  • 风格 — 1.5px 线条、圆角端点、有"手绘抖动"感、内填一抹暖色。
  • 载体 — SVG inline 字符串导出为 React 组件，按需 import，不走 iconfont。
  • 命名 — <Icon name="passport" />，可改色可改尺寸。
  • 动效 — tab 切换时图标轻微"draw-in"或"morph"动画。

肌理与装饰元素系统

让数字界面带一点"纸感、印章感、票根感"。

"行册"这个名字本身就在暗示一种纸质载体的气质。但当前的 UI 是干净的圆角卡片 + 渐变，肌理感非常弱（只有 itinerary 的票根 tab 表达出来一点）。可以系统性加一层"装饰图层"：

  • 纸纹底 — 给 surface 加非常淡的噪点/纤维纹理（background-image SVG noise）。
  • 邮戳印章 — 每个 trip 头部右上角自动盖一枚"目的地章"，复古复合。
  • 票根 tab — 你已经做了一版，再把它做完（撕口、虚线、序号）。
  • 手绘装饰 — 标题下的波浪线、空状态的手绘行李箱、按钮 hover 的手写下划线。
  • 边角磨损 — 重要卡片可加非常轻微的边角磨损/翘起，看起来"被翻阅过"。

AI 生成的剧场化动效

用户最该感觉到"AI 在为我工作"的地方，目前最没动。

现在的体验是：填表 → 跳回首页 → 看到一个灰色 bar 写"AI 生成中…" → 几十秒后变绿色"就绪"。整个过程毫无现场感。AI 是这个产品最大的差异化，结果它的视觉表达最弱。

建议设计一套 "剧场化"的多帧动效，分三个阶段：

  • 第 1 帧 · 思考 — 紫橙渐变 shimmer 横扫，显示"正在分析你的偏好…"等步骤性文案（不是 spinner）。
  • 第 2 帧 · 浮现 — 每生成一天，那一天的卡片从时间线"掉下来"到位（spring 缓动）。地点 name 用打字机效果一字一字浮现。
  • 第 3 帧 · 落地 — 全部生成完后，"AI 草稿就绪"以一个微微抖动的金色印章形式盖在右上角。点击展开预览。
  • 背景音 — 可选：很轻的"翻纸声"音效（用户可关）。
  • 错误态 — 不是冷冰冰红色，是"AI 写到一半放下笔"，配一句"换个想法重新试试？"
AI 不是后台进程，它是同行的旅伴。要让用户看见它在"写"。 — Design intent

AI 入口与采访式偏好表单

把"填表单"变成"和 AI 对话"。

现在的 AIPlanForm 是一个传统的 sheet：模型/节奏/出行人群/预算/其他偏好 — 5 个字段一次性铺开。这种表单在桌面端可以，但在小程序里显得很沉重，而且没"AI 在跟我聊"的感觉。

建议改成采访式（chat-like）多步对话，AI 一次问一个问题，用户点 chip 或自由输入回答：

  • Q1: "这次想要什么节奏？" → chips: 悠闲 / 平衡 / 紧凑
  • Q2: "和谁一起？" → chips: 独行 / 情侣 / 亲子 / 朋友 / 老人
  • Q3: "每人每天大概想花多少？" → 滑块或 chips：经济 / 中等 / 奢华
  • Q4: "有什么特别想做或不想做的吗？" → 自由输入（可跳过）
  • AI 入口 — 主页底部"+ 新建攻略"按钮可以裂变成两个：「自己填」「让 AI 帮我」，AI 那个带轻微粒子动效和呼吸光晕。
  • 每答一题 — 上一题缩为一行小总结，新问题滑入。最后展示"我听明白了，开始生成"的确认页。

首页：从列表到旅行册

"行册"这个名字值得一个比"一列卡片"更有想象力的首页。

现在的首页就是一个垂直的 trip 卡片列表 — 功能上没毛病，但完全没有"打开一本册子"的感觉。建议从下面 3 个隐喻里选一个深做（或做成主题切换的一部分）：

  • A · 明信片堆 — trips 像堆叠的明信片，轻微旋转、错落、可以"翻"。每张右上角自动盖目的地戳。
  • B · 旅行护照 — 整个首页是一本护照，每个 trip 是一页签证页，盖满目的地章。底部"+ 新页"。
  • C · 世界地图 — 一张极简地图作为主背景，trips 是地图上的 pin，点 pin 弹出 trip 卡片浮窗。"年度回顾"自然落地。

无论选哪个，都建议把主页头部从"行册 · 你的旅行，值得被好好记录"这种标语，改成期刊式 masthead：「行册 · No. 12 · 2026 春 · 共 5 段旅程」 — 让用户感到"我在写自己的杂志"。

行程的生命周期三态

出行前、出行中、出行后 — 同一个 trip，不同状态下应该长得完全不同。

这是我看完代码后最想推的一个方向。现在不管 trip 是"下周才出发"还是"正在路上"还是"三个月前回来的"，它在首页和详情页都长得一样 — 这是巨大的浪费。每个状态都有完全不同的用户意图：

  • 出行前 · 计划态 — 主视觉是倒计时「还有 12 天 · 北京」+ 清单完成度环 + 天气预告。首页这张卡片浮在最上。
  • 出行中 · 实时态 — 自动切到 Live 模式：当前 spot（深色卡片，强对比）+ 下一站时间 + 当地天气 + 一键叫车/导航。这就是产品的"高光时刻"。
  • 出行后 · 回顾态 — 自动生成「2026 春 · 北京五日」的回顾卡片：去过几个 spot、走了多少 km、花了多少钱、最贵的一餐是…。可分享。
  • 状态判断用 trip 的 startDate/endDate 自动切换，无需用户操作。
旅行不是一个矩形数据，是一段有起承转合的故事。UI 应该跟着这段故事走。 — Design intent

导航与层级重构

把"高频操作"沉底，把"信息密度"沉到内容里。

当前 trip 页面顶部有：trip 名 → AI 按钮 → 菜单 → AI 状态条 → 日期/人数 → 协作者条 → 4 tab → 内容 — 滚动还没开始就已经塞了 7 层。这造成两个问题：内容区域被压缩，关键操作（切换 tab）需要伸手到屏幕中上部。

建议：

  • tab 下沉 — 把 4 tab 从顶部移到底部 tab bar（攻略 / 地图 / 开销 / 清单），变成原生小程序底 tab 体验。
  • 顶部精简 — 只保留 trip 名 + 菜单。日期、人数、协作者折叠到副标题行，需要时点开。
  • AI 状态条从顶部固定改成右上角浮动徽章，不抢空间但仍可见。
  • + 加号从底部固定按钮，改成右下浮动 FAB，根据当前 tab 智能改变行为（在攻略 tab 加地点、在清单 tab 加物品）。
  • 头图 — 顶部 80rpx 渐变 banner 改成 200rpx 的目的地抽象封面图（地图剪影 / 目的地剪影）。

开销页的可视化升级

从一根水平条到一个有故事的"账本页"。

现在的 BudgetView 是：总开销 + 人均 + 一根 4 段堆叠条 + 按天列表。那根条几乎不可读（4 个颜色，肉眼难判断比例）。建议：

  • 主视觉换成环形图（donut），中心放总金额，四周是分类百分比。环形比条形更易读、更有"账本扉页"感。
  • 对比基线 — 显示"人均 ¥X · 比同行 Y 人少/多 N%"，给数字加意义。
  • 每日折线 — 在 BudgetView 加一行小折线图，看哪天花得最多/少，对应 trip 那天发生了啥。
  • "贵的一笔"高亮 — 自动标出最贵的一次餐饮、最贵的一晚住宿，做成"小高光卡"。
  • 多人 split — pax > 1 时，显示"AA 后每人差多少"，付款人自动平账（未来）。

分享 = 海报生成器

小程序里的"分享"是冷启动的金矿，但现在只是发了张文字卡。

现在的 ShareTypeSheet 只提供"只读分享 / 协作分享"两个文本分享 path。建议在此之上加一个杀手级功能："生成旅行海报" — 一张 9:16 的精美图，可保存到相册、发朋友圈：

  • 海报内容 — 目的地名（大）+ 日期 + 行册 logo + 几个核心数字（spots / km / 花费）+ 小程序码。
  • 风格变体 — 提供 3 种风格（电影海报 / 明信片 / 杂志封面），用户选。
  • 实现 — 微信小程序的 canvas API 或 wxml-to-canvas，本质就是把一个 wxml 渲染成图。
  • 触发时机 — 旅行结束自动弹"为这段旅程生成海报？"；也可以在分享菜单常驻入口。
  • 溢出价值 — 用户朋友圈晒图自带小程序码，是自然的获客渠道。
社交分享是 0 成本的增长。让用户「想晒」，比让用户「想用」更难也更值得。 — Design intent
