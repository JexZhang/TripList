# 第五章 · 分享系统

---

## 两种分享模式

| 模式 | 说明 | 接收方的操作 |
|------|------|------------|
| **只读分享** | 朋友可以看你的攻略，然后复制一份 | 点「复制这份攻略」→ 得到自己的独立副本 |
| **协作邀请** | 朋友加入后可以一起编辑 | 点「加入协作」→ 成为协作者 |

---

## Token 机制

分享链接的核心是一个随机 Token（就像一把「钥匙」）：

```
分享链接：
/pages/share/index?token=abc123def456&kind=readonly&tripId=trip001

token = abc123def456  ← 随机生成，7天有效
kind  = readonly       ← 分享类型
tripId= trip001        ← 对应的攻略
```

**为什么不直接传 tripId 就行，还要 Token？**

- 如果链接是 `/pages/share?tripId=trip001`，任何人猜到 ID 就能访问
- Token 是随机的 32 字符字符串，猜不到
- Token 有过期时间（7天），过期后链接失效

## 分享流程完整追踪

### 1. Owner 打开分享弹窗

```tsx
// src/pages/trip/index.tsx
const handleAction = async (action: TripAction) => {
  if (action === 'share') {
    resetShareRef(t.name)                  // 清空之前的 Token
    setShareReady({ readonly: false, collab: false })
    setShareOpen(true)
  }
}
```

### 2. ShareTypeSheet 出现，同时开始预生成 Token

```tsx
// src/components/ShareTypeSheet/index.tsx
useEffect(() => {
  if (!open) return

  // 打开时立刻开始生成两种 Token
  prepare('readonly')  // ← 先生成只读 Token
    .then(() => prepare('collab'))  // ← 再生成协作 Token
}, [open])
```

为什么提前生成？因为用户点「分享」按钮到微信分享弹窗出现，有时间差很短，来不及异步生成 Token。提前生成可以让分享更流畅。

### 3. 生成 Token

```tsx
// src/pages/trip/index.tsx
const prepareShare = async (kind: ShareKind) => {
  if (!isOwner) {
    Taro.showToast({ title: '仅 owner 可分享', icon: 'none' })
    return
  }

  try {
    // 调云函数生成 Token，并构造分享链接
    const payload = await buildShareMessage(t._id, t.name, kind)
    shareRef.byKind[kind] = { title: payload.title, path: payload.path }
    setShareReady(prev => ({ ...prev, [kind]: true }))  // 标记这个类型就绪了
  } catch (e) {
    Taro.showToast({ title: '生成分享失败', icon: 'error' })
  }
}
```

```typescript
// src/utils/share.ts
export async function buildShareMessage(tripId: string, tripName: string, kind: ShareKind) {
  const { token } = await cloud.createShareToken({ tripId, kind })

  return {
    title: `行册 · ${tripName}`,
    path: `/pages/share/index?token=${token}&kind=${kind}&tripId=${tripId}`,
  }
}
```

### 4. 分享按钮就绪

```tsx
// ShareTypeSheet 里的分享按钮
{ready.readonly ? (
  <Button openType='share' dataset={{ kind: 'readonly' }}>
    只读分享
  </Button>
) : (
  <View className='preparing'>准备中...</View>
)}
```

`openType='share'` 是微信小程序的特殊属性，点击时触发 `useShareAppMessage` 回调。

### 5. 微信分享回调

```tsx
// src/pages/trip/index.tsx
useShareAppMessage((options) => {
  const kind = options.target?.dataset?.kind  // 从按钮的 dataset 读 kind
  const picked = kind ? shareRef.byKind[kind] : null

  return picked || {
    title: shareRef.tripName ? `行册 · ${shareRef.tripName}` : '行册',
    path: '/pages/home/index',
  }
})
```

`shareRef` 是模块级变量（不是 state），在 React 渲染树之外，避免了 `useShareAppMessage` 闭包捕获旧值的问题。

```typescript
// src/utils/share.ts
export const shareRef = {
  tripName: '',
  byKind: { readonly: null, collab: null }
}

export function resetShareRef(tripName: string) {
  shareRef.tripName = tripName
  shareRef.byKind.readonly = null
  shareRef.byKind.collab = null
}
```

## 接收分享：Share 页面

朋友收到链接，点击进入 `/pages/share/index`：

```tsx
// src/pages/share/index.tsx
function SharePage() {
  const router = useRouter()
  const { token, kind, tripId } = router.params  // 从 URL 取参数

  const [tripMeta, setTripMeta] = useState(null)

  // 加载攻略基本信息（用于预览）
  useEffect(() => {
    getTrip(tripId).then(trip => {
      if (trip) setTripMeta(trip)
    })
  }, [tripId])

  const handleClone = async () => {
    // 只读分享：复制攻略
    const { newTripId } = await cloud.cloneTrip({ sourceTripId: tripId, token })
    Taro.redirectTo({ url: `/pages/trip/index?id=${newTripId}` })
  }

  const handleJoin = async () => {
    // 协作分享：加入协作
    const res = await cloud.joinCollab({ tripId, token })
    // 处理各种情况（已是owner、已加入、成功加入）
    Taro.redirectTo({ url: `/pages/trip/index?id=${tripId}` })
  }

  return (
    <View className='share-page'>
      {tripMeta && (
        <View className='share-card'>
          <Text className='sc-name'>{tripMeta.name}</Text>
          <Text className='sc-meta'>{tripMeta.days.length} 天 · {tripMeta.pax} 人</Text>
        </View>
      )}

      {kind === 'readonly' && (
        <View className='share-btn' onClick={handleClone}>
          复制这份攻略
        </View>
      )}

      {kind === 'collab' && (
        <View className='share-btn' onClick={handleJoin}>
          加入协作
        </View>
      )}
    </View>
  )
}
```

## 防止 Token 在二次分享中泄露

如果朋友收到攻略后，用「转发」按钮再分享给别人，不应该带上 Token（否则 Token 会一直传播）：

```tsx
// src/pages/share/index.tsx
useShareAppMessage(() => {
  return {
    title: tripMeta?.name ? `行册 · ${tripMeta.name}` : '行册',
    path: '/pages/home/index',  // ← 重定向到首页，不带 token 参数！
  }
})
```

## 小结

分享系统的设计要点：

| 要点 | 实现方式 |
|------|---------|
| 安全性 | 随机 32 字符 Token，非 tripId 直接暴露 |
| 有效期 | 7 天后 Token 过期 |
| 预生成 | 弹窗打开时就开始生成，避免用户等待 |
| 回调传参 | `shareRef` 模块变量（绕开闭包问题）|
| 防二次扩散 | 接收页的二次分享不带 Token |
