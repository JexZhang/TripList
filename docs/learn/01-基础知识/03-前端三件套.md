# 第一章 · 前端三件套：HTML、CSS、JavaScript

网页（和小程序）的界面由三种技术组成，就像建一栋房子：

| 技术 | 比喻 | 作用 |
|------|------|------|
| **HTML** | 房子的结构（墙、门、窗） | 定义界面有哪些元素 |
| **CSS** | 装修风格（颜色、大小、位置） | 控制元素的外观 |
| **JavaScript** | 电路系统（开灯、开门） | 实现交互逻辑 |

---

## 一、HTML：描述「有什么」

HTML 用「标签」来描述元素：

```html
<div>这是一个容器</div>
<text>这是一段文字</text>
<input placeholder="请输入..." />
<button>点我</button>
```

在行迹的代码里，用的是 Taro 提供的组件（本质是一样的，只是换了名字）：

```tsx
// 行迹代码里的 "HTML"（来自 SpotCard 组件）
<View className='spot-card'>          {/* 相当于 <div> */}
  <Text className='sc-name'>德基广场</Text>    {/* 相当于 <span> */}
  <Text className='sc-price'>¥200</Text>
</View>
```

> **小程序里没有 `<div>`、`<span>`**  
> 小程序用 `<View>`（相当于 div）和 `<Text>`（相当于 span），这是微信的规定。  
> Taro 帮我们处理了这些差异。

## 二、CSS：描述「长什么样」

CSS 通过「选择器」找到元素，然后设置样式：

```css
/* 找到 class 叫 spot-card 的元素，设置它的样式 */
.spot-card {
  background: white;      /* 背景色白色 */
  padding: 24px;          /* 内边距 24px（四周留空间）*/
  border-radius: 12px;    /* 圆角 */
  box-shadow: 0 2px 8px rgba(0,0,0,0.1);  /* 阴影 */
}
```

行迹使用的是 **SCSS**（CSS 的升级版），可以嵌套写，更清晰：

```scss
// 来自 src/views/ItineraryView/index.scss
.spot-card {
  padding: 24rpx 24rpx 20rpx;
  background: var(--surface);    // 使用「变量」，不是写死的颜色
  border-radius: var(--r-md);
  box-shadow: var(--shadow-sm);

  .sc-name {          // 嵌套：.spot-card 里面的 .sc-name
    font-size: 30rpx;
    font-weight: 600;
    color: var(--ink);
  }
}
```

### CSS 最常用属性

```css
/* 布局 */
display: flex;          /* 弹性布局（最常用） */
flex-direction: column; /* flex 方向：纵向 */
align-items: center;    /* 交叉轴对齐 */
justify-content: space-between; /* 主轴对齐 */

/* 尺寸 */
width: 100%;    /* 宽度 */
height: 50vh;   /* 高度（50% 视口高度）*/
padding: 16px;  /* 内边距 */
margin: 8px 0;  /* 外边距 */

/* 视觉 */
background: #FF7A2E;  /* 背景色 */
color: white;         /* 文字颜色 */
font-size: 28rpx;     /* 字体大小 */
border-radius: 12px;  /* 圆角 */

/* 定位 */
position: fixed;      /* 固定在屏幕上，不随页面滚动 */
position: absolute;   /* 相对于父元素定位 */
z-index: 100;         /* 层叠顺序，越大越在上面 */
```

> **rpx 是什么？**  
> 小程序专用单位，「响应式像素」。在 375px 宽的屏幕上，`1rpx = 0.5px`。  
> 使用 rpx 可以让界面在不同屏幕尺寸上自动适配大小。

### 行迹的设计变量系统

行迹在 `src/app.scss` 里定义了一套「变量」，所有颜色、圆角、阴影都通过变量名引用：

```scss
page {
  --bg: #FFF4E6;          // 背景色（暖橘米色）
  --surface: #FFFFFF;     // 卡片背景（白）
  --ink: #2B1A10;         // 文字主色（深棕）
  --accent: #FF7A2E;      // 品牌橙色

  --r-sm: 16rpx;          // 小圆角
  --r-md: 22rpx;          // 中圆角
  --r-pill: 999rpx;       // 胶囊形状

  --shadow-sm: 0 4rpx 14rpx rgba(255, 122, 46, 0.10);
}
```

这样做的好处：如果要改主题颜色，只改一个地方，全局都变。

## 三、JavaScript：描述「做什么」

JavaScript（简称 JS）让界面能响应用户操作：

```javascript
// 最基础的：声明变量、做计算
let count = 0
count = count + 1
console.log(count)  // 输出到控制台：1

// 函数：把一段逻辑打包
function greet(name) {
  return '你好，' + name
}
greet('张三')  // → '你好，张三'

// 条件判断
if (count > 5) {
  console.log('超过 5 了！')
} else {
  console.log('还没到 5')
}

// 数组操作
const fruits = ['苹果', '香蕉', '橙子']
fruits.map(f => f + '🍎')  // → ['苹果🍎', '香蕉🍎', '橙子🍎']
fruits.filter(f => f !== '香蕉')  // → ['苹果', '橙子']
```

### 行迹里的 JS：处理行程数据

```tsx
// 计算总花费（来自 BudgetView）
const total = trips
  .flatMap(day => day.spots)       // 展开所有天的地点
  .filter(spot => spot.price > 0)  // 只看有价格的
  .reduce((sum, spot) => sum + spot.price, 0)  // 求和
```

### 异步操作

网络请求需要时间，JS 用「异步」来处理：

```javascript
// 老写法（回调地狱）
fetch('/api/trips', function(trips) {
  saveToState(trips)
})

// 现代写法（async/await），行迹里用的这个
async function loadTrips() {
  const trips = await listMyTrips(openid)  // 等待数据回来
  setTrips(trips)                          // 然后更新界面
}
```

`await` 就像说「等一下，等这件事做完了再继续」。

## 四、TypeScript：JS 的升级版

行迹用的是 **TypeScript**（简称 TS），在 JS 基础上加了「类型」：

```typescript
// 普通 JS：没有类型约束，容易出错
let name = '南京'
name = 123  // 不报错，但逻辑上是错的

// TypeScript：有类型检查
let name: string = '南京'
name = 123  // ❌ 报错！name 必须是字符串
```

行迹里定义了所有数据的「类型」，在 `src/types/trip.ts`：

```typescript
// 一个「地点」长什么样
interface Spot {
  id: string           // 唯一标识符
  type: 'spot' | 'hotel' | 'meal' | 'transport'  // 只能是这四种
  name: string         // 地点名
  lat?: number         // 纬度（问号表示可以没有）
  lng?: number         // 经度
  price?: number       // 价格
  note?: string        // 备注
}

// 一天的行程
interface Day {
  id: string
  date: string         // 格式：'2024-05-29'
  spots: Spot[]        // 这一天的地点列表（数组）
}
```

TypeScript 的好处：
- 写代码时，编辑器能自动提示
- 类型不匹配会报错，避免运行时崩溃
- 读代码更清晰，知道每个变量是什么

## 五、JSX：HTML 和 JS 的结合

React（行迹用的框架）创造了 **JSX** 语法，让你在 JS 里写「HTML」：

```tsx
// 这就是 JSX
function SpotCard({ spot }) {
  return (
    <View className='spot-card'>
      <Text>{spot.name}</Text>                    {/* {} 里可以写 JS 表达式 */}
      {spot.price && <Text>¥{spot.price}</Text>}  {/* 条件渲染 */}
      {spot.type === 'hotel' && <Text>🏨</Text>}  {/* 另一种条件 */}
    </View>
  )
}
```

规则：
- `{}` 里写 JavaScript 表达式
- HTML 属性用 `className` 不用 `class`（因为 class 是 JS 关键字）
- 标签必须关闭（`<Text>` 必须有 `</Text>` 或 `<Text />`）

## 六、把三件套联系起来

看一个完整的例子——「添加地点」按钮：

```tsx
// src/views/ItineraryView/index.tsx（简化版）
function ItineraryView() {
  const [searchOpen, setSearchOpen] = useState(false)  // JS：状态

  return (
    <View>
      {/* HTML：结构 */}
      <View className='itin-add-spot' onClick={() => setSearchOpen(true)}>
        + 添加地点
      </View>

      {/* 条件渲染搜索弹窗 */}
      <SpotSearch open={searchOpen} onClose={() => setSearchOpen(false)} />
    </View>
  )
}
```

```scss
/* CSS：样式 */
.itin-add-spot {
  padding: 28rpx;
  text-align: center;
  background: var(--surface-2);
  border: 2rpx dashed var(--accent-soft);
  border-radius: var(--r-md);
  color: var(--accent);
  font-weight: 600;
}
```

效果：一个有虚线边框、橙色文字的按钮，点击会弹出搜索框。
