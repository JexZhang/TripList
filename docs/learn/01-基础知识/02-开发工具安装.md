# 第一章 · 开发工具安装

## 你需要安装的东西

| 工具 | 用途 | 地址 |
|------|------|------|
| Node.js (≥18) | 运行 JavaScript 的环境 | https://nodejs.org |
| VS Code | 代码编辑器 | https://code.visualstudio.com |
| 微信开发者工具 | 预览/调试小程序 | https://developers.weixin.qq.com/miniprogram/dev/devtools |
| Git | 版本控制（记录代码变化历史） | 大多数 Mac 自带；Windows 从 https://git-scm.com 下 |

---

## Step 1：安装 Node.js

Node.js 是让你能在电脑上运行 JavaScript 代码的环境（浏览器外面）。

1. 打开 https://nodejs.org
2. 下载 **LTS** 版本（长期支持版，更稳定）
3. 一路点「下一步」安装

验证安装成功：打开终端（Mac 用 Terminal，Windows 用 PowerShell），输入：

```bash
node --version
# 应该输出类似 v20.x.x

npm --version
# 应该输出类似 10.x.x
```

> 💡 **什么是终端？**  
> 终端就是「命令行界面」，你可以输入命令让电脑做事，而不是用鼠标点击。  
> Mac：按 `Command + 空格`，搜索「Terminal」  
> Windows：按 `Win + R`，输入 `powershell`

---

## Step 2：安装 VS Code

VS Code 是最流行的代码编辑器，免费、功能强大。

1. 下载安装后，安装这些扩展（在左侧栏点「扩展」图标，搜索名字）：
   - **ESLint** — 检查代码规范
   - **Prettier** — 自动格式化代码
   - **TypeScript** — TypeScript 支持（一般已内置）

---

## Step 3：安装微信开发者工具

这是微信官方提供的工具，用来预览你的小程序效果。

1. 进入 https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html
2. 下载对应系统的安装包
3. 安装后，用微信扫码登录

---

## Step 4：克隆项目代码

如果项目在 GitHub 上，用 Git 下载：

```bash
git clone https://github.com/JexZhang/TripList.git
cd TripList
```

如果代码已经在本地了，跳过这步。

---

## Step 5：安装项目依赖

进入项目目录，安装所有依赖包：

```bash
npm install
```

这会读取 `package.json` 文件，自动下载项目需要的所有库。  
等待几分钟，看到「added N packages」就完成了。

> ❓ **什么是依赖包？**  
> 就像做菜需要买食材，写代码需要用别人写好的库。  
> `npm install` 就是去「超市」（npm 仓库）把你需要的「食材」下载回来。  
> 这些文件存在 `node_modules/` 文件夹里（通常有几百 MB）。

---

## Step 6：编译项目

行迹用 Taro 写，需要先编译成小程序能识别的格式：

```bash
npm run dev:weapp
```

这会：
1. 把 TypeScript 编译成 JavaScript
2. 把 SCSS 编译成 CSS
3. 把 React/Taro 代码转换成小程序格式
4. 把结果输出到 `dist/` 文件夹

看到如下输出说明成功了：
```
Compiled successfully
```

编译后会持续监听文件变化（你改代码它自动重新编译）。

---

## Step 7：在开发者工具里预览

1. 打开**微信开发者工具**
2. 点「导入项目」
3. 目录选择 `TripList/dist/`（注意是 `dist` 不是项目根目录）
4. AppID 填写 `wx833348388eb83bc1`（或者用测试 ID）
5. 点确定，就能看到小程序界面了！

## 常见安装问题

### npm install 很慢或失败

换成国内镜像：
```bash
npm config set registry https://registry.npmmirror.com
npm install
```

### 找不到 `dist` 文件夹

说明编译还没完成或失败了。检查 `npm run dev:weapp` 的输出，看有没有报错信息。

### 微信开发者工具提示「AppID 无效」

先选「测试号」，可以用来本地开发，功能略有限制（云开发无法使用）。

### 云函数相关功能不工作

云开发功能只有用真实的 AppID 才能测试。本地开发时可以先跳过，专注界面部分。

## 项目的 npm 命令一览

打开 `package.json` 可以看到所有命令：

```json
{
  "scripts": {
    "dev:weapp": "taro build --type weapp --watch",  // 开发模式（持续编译）
    "build:weapp": "taro build --type weapp",         // 生产构建
    "lint": "eslint src --ext .ts,.tsx",              // 检查代码规范
    "lint:style": "stylelint src/**/*.scss"           // 检查样式规范
  }
}
```

## 目录结构预览

安装完成后，你的项目目录应该是这样的：

```
TripList/
├── src/              ← 你写代码的地方
├── dist/             ← 编译后的输出（微信开发者工具打开这里）
├── cloudfunctions/   ← 云函数（后端逻辑）
├── docs/             ← 文档（你正在读的就在这）
├── node_modules/     ← 依赖包（不要手动改这里的东西）
├── package.json      ← 项目配置和依赖列表
└── tsconfig.json     ← TypeScript 配置
```
