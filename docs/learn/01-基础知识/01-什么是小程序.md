# 第一章 · 什么是小程序

## 1. 从你的手机说起

打开微信，点击右上角「+」→「搜索」，输入「行册」——如果找到了，点击就能直接用。不需要在 App Store 下载，不需要安装，用完可以直接关掉。这就是**小程序**。

小程序的核心特点：
- ✅ 无需下载安装（「用完即走」）
- ✅ 运行在微信内部，不是独立 App
- ✅ 可以调用手机硬件（相机、地理位置等）
- ✅ 有微信账号就有登录，不用注册

---

## 2. 小程序 vs 普通 App vs 网页

| 对比项 | 普通 App | 网页 | 小程序 |
|--------|---------|------|--------|
| 需要安装 | ✅ 是 | ❌ 否 | ❌ 否 |
| 能调用手机功能 | ✅ 全部 | ⚠️ 有限 | ✅ 大部分 |
| 用微信账号登录 | ❌ 不行 | ❌ 不行 | ✅ 直接用 |
| 开发成本 | 高（iOS+Android分别写） | 中 | 中低 |
| 分发方式 | App Store | 网址链接 | 微信内搜索/扫码 |

**行册**选择做小程序，主要是因为：
1. 用户不需要安装就能用
2. 旅行时分享给朋友很方便（发微信链接就行）
3. 可以直接用微信身份，不用再注册账号

## 3. 小程序怎么工作

用最简单的比喻理解：

```
你的手机（微信）         服务器（云端）
┌──────────────┐        ┌──────────────┐
│              │  请求  │              │
│  你看到的界面 │ ──→── │  数据库、逻辑  │
│              │  响应  │              │
│              │ ──←── │              │
└──────────────┘        └──────────────┘
      前端                    后端
```

- **前端**：你眼睛看到的、手指触碰的界面。由代码画出来，运行在你的手机上。
- **后端**：存数据的地方。你的行程信息、好友信息，都存在服务器里。
- 两端通过**网络请求**通信。

行册的后端用的是微信自己提供的「**云开发**」——相当于微信帮你搭好了服务器，你只需要写业务逻辑。

## 4. 行册的页面结构

打开行册，你会经历这样的页面流程：

```
启动
 │
 ↓
首页（攻略册）          ← 看已有的攻略列表
 │
 ├── 点「新建」→ 新建攻略页    ← 填写名称、日期、目的地
 │
 └── 点某个攻略→ 攻略详情页   ← 这是核心页面！
         │
         ├── 攻略 tab（行程地点）
         ├── 开销 tab（花费统计）
         ├── 清单 tab（打包清单）
         └── 地图 tab（地点地图）
```

每个「页面」都是一个独立的代码文件，存在 `src/pages/` 文件夹里。

## 5. 代码长什么样

先不用看懂，感受一下：

```tsx
// 这是「首页」的核心代码片段（src/pages/home/index.tsx）

export default function HomePage() {
  const [trips, setTrips] = useState([])   // 攻略列表，初始为空
  
  useEffect(() => {
    // 页面加载时，从云端取攻略数据
    listMyTrips(openid).then(data => setTrips(data))
  }, [])

  return (
    <View className='home'>
      {trips.map(trip => (
        <TripCard key={trip._id} trip={trip} />   // 每条攻略显示一张卡片
      ))}
    </View>
  )
}
```

这段代码做了三件事：
1. 声明「攻略列表」这个数据（`useState`）
2. 页面加载时从云端拉数据（`useEffect`）
3. 把数据渲染成一个个卡片（`return` 里的 JSX）

## 6. 开发小程序需要什么知识

你需要学习的技能栈，从底层到上层：

```
TypeScript（JavaScript 的升级版）
         ↑
    React（界面框架）
         ↑
    Taro（跨端框架，把 React 代码编译成小程序）
         ↑
   微信云开发（数据库、云函数）
         ↑
     高德地图 API（地图相关）
```

听起来很多？别慌。这本手册会一步步解释每一层是什么，结合行册的实际代码来讲。

---

## 7. 小程序的限制（和普通网页开发的不同）

知道这些，避免后面踩坑：

| 限制 | 说明 |
|------|------|
| 包体大小限制 | 主包最大 2MB（所以要注意图片不能直接放代码里） |
| 没有 `window`/`document` | 不能用浏览器特有的 API |
| 网络请求限制 | 只能请求备案过的域名 |
| 不能动态执行代码 | 不能用 `eval()` |
| 渲染引擎不同 | iOS 用 JavaScriptCore，Android 用 V8，小区别 |

行册用 **Taro** 框架来写代码，这样你写的是标准 React 代码，Taro 负责把它转换成小程序能运行的格式。
