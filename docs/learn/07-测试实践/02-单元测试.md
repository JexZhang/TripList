# 第七章 · 单元测试

单元测试是最容易上手的测试类型——给一个函数输入，验证它的输出。

---

## 测试框架：Jest 基础语法

```javascript
// 最基础的测试结构
test('测试描述', () => {
  // 准备数据
  const input = 1234

  // 执行被测代码
  const result = fmtCurrency(input)

  // 断言结果
  expect(result).toBe('¥1,234')
})
```

### 常用断言

```javascript
expect(value).toBe('精确相等')           // 基础类型（字符串/数字/布尔）
expect(value).toEqual({ a: 1 })          // 深度相等（对象/数组）
expect(value).toBeTruthy()               // 值为真
expect(value).toBeFalsy()                // 值为假
expect(value).toBeNull()                 // 是 null
expect(value).toBeUndefined()            // 是 undefined
expect(value).toContain('子字符串')       // 字符串包含
expect(array).toContain('元素')          // 数组包含
expect(array).toHaveLength(3)            // 数组长度
expect(fn).toThrow('错误信息')           // 函数抛出异常

// 数字比较
expect(5).toBeGreaterThan(3)
expect(5).toBeLessThanOrEqual(5)
expect(3.14).toBeCloseTo(3.1415, 2)     // 浮点数近似相等（保留2位）
```

### describe：给测试分组

```javascript
describe('fmtCurrency', () => {
  test('整数金额', () => {
    expect(fmtCurrency(1234)).toBe('¥1,234')
  })

  test('零元', () => {
    expect(fmtCurrency(0)).toBe('¥0')
  })

  test('大金额', () => {
    expect(fmtCurrency(1000000)).toBe('¥1,000,000')
  })
})
```

## 给 format.ts 写测试

`src/utils/format.ts` 是纯函数，最适合单元测试：

```typescript
// src/utils/format.test.ts
import { fmtCurrency, fmtDate, fmtDateShort, daysBetween } from './format'

describe('fmtCurrency', () => {
  test('正常金额加千分位和符号', () => {
    expect(fmtCurrency(1234)).toBe('¥1,234')
  })

  test('零元', () => {
    expect(fmtCurrency(0)).toBe('¥0')
  })

  test('大金额有多个千分位', () => {
    expect(fmtCurrency(1234567)).toBe('¥1,234,567')
  })

  test('自定义货币符号', () => {
    expect(fmtCurrency(100, '$')).toBe('$100')
  })
})

describe('fmtDate', () => {
  test('默认格式', () => {
    expect(fmtDate('2024-05-29')).toBe('2024-05-29')
  })

  test('自定义格式', () => {
    expect(fmtDate('2024-05-29', 'MM/DD')).toBe('05/29')
  })
})

describe('fmtDateShort', () => {
  test('输出 MM.DD 格式', () => {
    expect(fmtDateShort('2024-05-29')).toBe('05.29')
  })

  test('一月份补零', () => {
    expect(fmtDateShort('2024-01-05')).toBe('01.05')
  })
})

describe('daysBetween', () => {
  test('同一天 = 1 天', () => {
    expect(daysBetween('2024-05-29', '2024-05-29')).toBe(1)
  })

  test('相邻两天 = 2 天', () => {
    expect(daysBetween('2024-05-29', '2024-05-30')).toBe(2)
  })

  test('跨月', () => {
    expect(daysBetween('2024-05-29', '2024-06-02')).toBe(5)
  })
})
```

## 给 trip-helpers.ts 写测试

```typescript
// src/utils/trip-helpers.test.ts
import { seedDays, buildNewTrip, destinationLabel, tripSummary } from './trip-helpers'

describe('seedDays', () => {
  test('生成正确数量的天数', () => {
    const days = seedDays('2024-05-29', '2024-06-02')
    expect(days).toHaveLength(5)  // 5月29 到 6月2 = 5天
  })

  test('同一天生成 1 天', () => {
    const days = seedDays('2024-05-29', '2024-05-29')
    expect(days).toHaveLength(1)
  })

  test('每天有空的 spots 数组', () => {
    const days = seedDays('2024-05-29', '2024-05-30')
    expect(days[0].spots).toEqual([])
    expect(days[1].spots).toEqual([])
  })

  test('日期按顺序排列', () => {
    const days = seedDays('2024-05-29', '2024-05-31')
    expect(days[0].date).toBe('2024-05-29')
    expect(days[1].date).toBe('2024-05-30')
    expect(days[2].date).toBe('2024-05-31')
  })

  test('每天有唯一 id', () => {
    const days = seedDays('2024-05-29', '2024-05-31')
    const ids = days.map(d => d.id)
    const uniqueIds = new Set(ids)
    expect(uniqueIds.size).toBe(ids.length)  // 没有重复 id
  })
})

describe('destinationLabel', () => {
  test('一个目的地', () => {
    const dests = [{ name: '南京', adcode: '320100', lat: 32, lng: 118 }]
    expect(destinationLabel(dests)).toBe('南京')
  })

  test('多个目的地用 · 分隔', () => {
    const dests = [
      { name: '南京', adcode: '320100', lat: 32, lng: 118 },
      { name: '杭州', adcode: '330100', lat: 30, lng: 120 },
    ]
    expect(destinationLabel(dests)).toBe('南京 · 杭州')
  })

  test('空数组返回空字符串', () => {
    expect(destinationLabel([])).toBe('')
  })
})

describe('tripSummary', () => {
  test('正确生成摘要', () => {
    const result = tripSummary('2024-05-29', '2024-06-01', 3)
    expect(result).toBe('4 天 · 3 人')
  })
})
```

## 给 MapView helpers 写测试

```typescript
// src/views/MapView/helpers.test.ts
import { encodeMarkerId, decodeMarkerId, dayColor, collectLocated } from './helpers'

describe('encodeMarkerId / decodeMarkerId', () => {
  test('编码后能解码回来', () => {
    const original = { dayIdx: 2, spotIdx: 5 }
    const encoded = encodeMarkerId(original.dayIdx, original.spotIdx)
    const decoded = decodeMarkerId(encoded)
    expect(decoded).toEqual(original)
  })

  test('第0天第0个地点 = id 0', () => {
    expect(encodeMarkerId(0, 0)).toBe(0)
  })

  test('第1天第0个地点 = id 1000', () => {
    expect(encodeMarkerId(1, 0)).toBe(1000)
  })

  test('不同位置编码不同', () => {
    const id1 = encodeMarkerId(0, 1)
    const id2 = encodeMarkerId(1, 0)
    expect(id1).not.toBe(id2)
  })
})

describe('dayColor', () => {
  test('第0天返回字符串颜色', () => {
    expect(typeof dayColor(0)).toBe('string')
    expect(dayColor(0)).toMatch(/^#[0-9A-Fa-f]{6}$/)  // 合法的颜色格式
  })

  test('超过颜色数量后循环', () => {
    // 7 种颜色循环，所以第0天和第7天颜色相同
    expect(dayColor(0)).toBe(dayColor(7))
    expect(dayColor(1)).toBe(dayColor(8))
  })
})

describe('collectLocated', () => {
  const days = [
    {
      id: 'd1', date: '2024-05-29',
      spots: [
        { id: 's1', name: '玄武湖', lat: 32.07, lng: 118.80, type: 'spot' as const },
        { id: 's2', name: '没坐标', lat: undefined, lng: undefined, type: 'spot' as const },
      ],
      weather: null,
    },
    {
      id: 'd2', date: '2024-05-30',
      spots: [
        { id: 's3', name: '夫子庙', lat: 32.01, lng: 118.78, type: 'spot' as const },
      ],
      weather: null,
    },
  ]

  test('只收集有坐标的地点', () => {
    const result = collectLocated(days)
    expect(result).toHaveLength(2)  // s2 没坐标，不收集
  })

  test('按 dayIdx + spotIdx 记录位置', () => {
    const result = collectLocated(days)
    expect(result[0]).toMatchObject({ dayIdx: 0, spotIdx: 0 })  // 玄武湖
    expect(result[1]).toMatchObject({ dayIdx: 1, spotIdx: 0 })  // 夫子庙
  })

  test('onlyDayIdx 只收集指定天', () => {
    const result = collectLocated(days, 1)  // 只要第1天
    expect(result).toHaveLength(1)
    expect(result[0]).toMatchObject({ dayIdx: 1 })
  })
})
```

## 给 Reducer 写测试

Reducer 是纯函数（输入旧状态 + action，输出新状态），非常适合测试：

```typescript
// src/store/trip-store.test.ts
import { reducer } from './trip-store'  // 需要把 reducer 导出

const makeTrip = (overrides = {}) => ({
  _id: 'trip-001',
  _openid: 'user-001',
  name: '测试攻略',
  pax: 2,
  startDate: '2024-05-29',
  endDate: '2024-05-31',
  days: [
    {
      id: 'day-001',
      date: '2024-05-29',
      spots: [],
      weather: null,
    },
  ],
  packing: [],
  destinations: [],
  collaborators: [],
  collaboratorOpenids: [],
  ownerOpenid: 'user-001',
  ownerNickname: '测试用户',
  ownerAvatarUrl: '',
  createdAt: '2024-05-28T00:00:00Z',
  updatedAt: '2024-05-28T00:00:00Z',
  ...overrides,
})

const initialState = { trip: null, loading: true, error: null }

describe('tripReducer', () => {
  describe('SET_TRIP', () => {
    test('设置攻略数据', () => {
      const trip = makeTrip()
      const state = reducer(initialState, { type: 'SET_TRIP', trip })
      expect(state.trip).toEqual(trip)
      expect(state.loading).toBe(false)
    })
  })

  describe('ADD_SPOT', () => {
    test('向指定天添加地点', () => {
      const trip = makeTrip()
      const state0 = { trip, loading: false, error: null }
      const newSpot = { id: 'spot-001', type: 'spot' as const, name: '玄武湖' }

      const state1 = reducer(state0, {
        type: 'ADD_SPOT',
        dayId: 'day-001',
        spot: newSpot,
      })

      expect(state1.trip!.days[0].spots).toHaveLength(1)
      expect(state1.trip!.days[0].spots[0]).toEqual(newSpot)
    })

    test('不影响其他天', () => {
      const trip = makeTrip({
        days: [
          { id: 'day-001', date: '2024-05-29', spots: [], weather: null },
          { id: 'day-002', date: '2024-05-30', spots: [], weather: null },
        ]
      })
      const state0 = { trip, loading: false, error: null }

      const state1 = reducer(state0, {
        type: 'ADD_SPOT',
        dayId: 'day-001',
        spot: { id: 'spot-001', type: 'spot' as const, name: '玄武湖' },
      })

      // day-002 的 spots 不受影响
      expect(state1.trip!.days[1].spots).toHaveLength(0)
    })
  })

  describe('DELETE_SPOT', () => {
    test('删除指定地点', () => {
      const trip = makeTrip({
        days: [{
          id: 'day-001', date: '2024-05-29', weather: null,
          spots: [
            { id: 'spot-001', type: 'spot' as const, name: '玄武湖' },
            { id: 'spot-002', type: 'spot' as const, name: '夫子庙' },
          ]
        }]
      })
      const state0 = { trip, loading: false, error: null }

      const state1 = reducer(state0, {
        type: 'DELETE_SPOT',
        dayId: 'day-001',
        spotId: 'spot-001',
      })

      expect(state1.trip!.days[0].spots).toHaveLength(1)
      expect(state1.trip!.days[0].spots[0].name).toBe('夫子庙')
    })
  })

  describe('UPDATE_SPOT', () => {
    test('更新地点的指定字段', () => {
      const trip = makeTrip({
        days: [{
          id: 'day-001', date: '2024-05-29', weather: null,
          spots: [{ id: 'spot-001', type: 'spot' as const, name: '玄武湖', price: 0 }]
        }]
      })
      const state0 = { trip, loading: false, error: null }

      const state1 = reducer(state0, {
        type: 'UPDATE_SPOT',
        dayId: 'day-001',
        spotId: 'spot-001',
        patch: { price: 50, note: '需要提前购票' },
      })

      const updatedSpot = state1.trip!.days[0].spots[0]
      expect(updatedSpot.price).toBe(50)
      expect(updatedSpot.note).toBe('需要提前购票')
      expect(updatedSpot.name).toBe('玄武湖')  // 未修改的字段不变
    })
  })

  describe('不可变性', () => {
    test('Reducer 不修改原始 state', () => {
      const trip = makeTrip()
      const state0 = { trip, loading: false, error: null }
      const originalDays = state0.trip!.days

      reducer(state0, {
        type: 'ADD_SPOT',
        dayId: 'day-001',
        spot: { id: 'spot-001', type: 'spot' as const, name: '玄武湖' },
      })

      // 原始 state 的 days 不应该被修改
      expect(state0.trip!.days).toBe(originalDays)
      expect(state0.trip!.days[0].spots).toHaveLength(0)
    })
  })
})
```

## 运行测试

```bash
# 运行全部测试
npx jest

# 运行单个文件
npx jest src/utils/format.test.ts

# 监听模式（文件变化自动重跑）
npx jest --watch

# 查看覆盖率报告
npx jest --coverage
```

覆盖率报告长这样：

```
File          | % Stmts | % Branch | % Funcs | % Lines
--------------|---------|----------|---------|--------
format.ts     |   100   |   85.71  |   100   |   100
trip-helpers.ts|   92.3 |   75.00  |   100   |   92.3
```

- **Stmts**：语句覆盖率（有多少行被执行到）
- **Branch**：分支覆盖率（`if/else` 的每个分支都测到了吗）
- **Funcs**：函数覆盖率
- **Lines**：行覆盖率

100% 覆盖率不是目标，但 80%+ 是个不错的参考值。

## 测试的几个原则

### 1. 一个测试只验证一件事

```javascript
// ❌ 一个测试测太多
test('fmtCurrency', () => {
  expect(fmtCurrency(1234)).toBe('¥1,234')
  expect(fmtCurrency(0)).toBe('¥0')
  expect(fmtCurrency(1000000)).toBe('¥1,000,000')
})
// 如果第一个失败，后两个根本不会跑

// ✅ 拆开来，每个测试独立
test('整数有千分位', () => { expect(fmtCurrency(1234)).toBe('¥1,234') })
test('零元', () => { expect(fmtCurrency(0)).toBe('¥0') })
test('百万金额', () => { expect(fmtCurrency(1000000)).toBe('¥1,000,000') })
```

### 2. 测试边界条件

边界条件是最容易出 Bug 的地方：
- 空数组 `[]`
- `0`、`null`、`undefined`
- 极大值、极小值
- 只有一个元素的数组

### 3. 测试要稳定

测试不应该因为时间、随机数等因素而时灵时不灵：

```javascript
// ❌ 不稳定：依赖当前时间
test('今天的日期', () => {
  expect(fmtDate(new Date())).toBe('2024-05-29')  // 明天就失败了
})

// ✅ 稳定：用固定值
test('格式化给定日期', () => {
  expect(fmtDate('2024-05-29')).toBe('2024-05-29')  // 永远通过
})
```
