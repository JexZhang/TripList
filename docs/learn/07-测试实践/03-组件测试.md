# 第七章 · 组件测试

组件测试验证 React 组件的渲染结果和交互行为，比单元测试更接近用户的真实使用场景。

---

## 工具介绍

```
@testing-library/react    — 渲染组件、查询元素
@testing-library/jest-dom — 扩展断言（toBeInTheDocument 等）
@testing-library/user-event — 模拟用户交互（点击、输入等）
```

安装：
```bash
npm install --save-dev @testing-library/react @testing-library/jest-dom @testing-library/user-event
```

---

## 核心理念：从用户角度测试

Testing Library 的核心理念是：**测试要模拟用户行为，而不是实现细节**。

```javascript
// ❌ 测试实现细节（脆弱，内部重构就失败）
expect(component.state.searchOpen).toBe(true)

// ✅ 测试用户可见的结果（健壮）
expect(screen.getByText('添加地点')).toBeInTheDocument()
```

## 基础：渲染和查询

```tsx
import { render, screen } from '@testing-library/react'

test('渲染地点卡片', () => {
  const spot = {
    id: 'spot-001',
    type: 'spot' as const,
    name: '玄武湖',
    price: 50,
    note: '需要购票',
  }

  render(<SpotCard spot={spot} onClick={() => {}} />)

  // 查询元素
  expect(screen.getByText('玄武湖')).toBeInTheDocument()
  expect(screen.getByText('¥50')).toBeInTheDocument()
  expect(screen.getByText('需要购票')).toBeInTheDocument()
})
```

### 常用查询方法

```javascript
// getBy —— 找不到就报错（推荐用于必定存在的元素）
screen.getByText('玄武湖')
screen.getByRole('button', { name: '保存' })
screen.getByPlaceholderText('搜索地点')
screen.getByTestId('spot-card')  // 需要在 JSX 里加 data-testid="spot-card"

// queryBy —— 找不到返回 null（用于验证元素不存在）
expect(screen.queryByText('错误信息')).not.toBeInTheDocument()

// findBy —— 异步查询，等待元素出现（用于异步渲染）
const result = await screen.findByText('搜索结果')
```

## 测试 SpotCard 组件

```tsx
// src/views/ItineraryView/SpotCard.test.tsx
import { render, screen, fireEvent } from '@testing-library/react'
import SpotCard from './SpotCard'
import type { Spot } from '../../types/trip'

const makeSpot = (overrides: Partial<Spot> = {}): Spot => ({
  id: 'spot-001',
  type: 'spot',
  name: '玄武湖',
  ...overrides,
})

describe('SpotCard', () => {
  test('显示地点名称', () => {
    render(<SpotCard spot={makeSpot()} onClick={() => {}} />)
    expect(screen.getByText('玄武湖')).toBeInTheDocument()
  })

  test('有价格时显示金额', () => {
    render(<SpotCard spot={makeSpot({ price: 50 })} onClick={() => {}} />)
    expect(screen.getByText('¥50')).toBeInTheDocument()
  })

  test('价格为 0 时不显示', () => {
    render(<SpotCard spot={makeSpot({ price: 0 })} onClick={() => {}} />)
    expect(screen.queryByText('¥0')).not.toBeInTheDocument()
  })

  test('有备注时显示备注', () => {
    render(<SpotCard spot={makeSpot({ note: '记得带望远镜' })} onClick={() => {}} />)
    expect(screen.getByText('记得带望远镜')).toBeInTheDocument()
  })

  test('没有备注和价格时不渲染底部区域', () => {
    render(<SpotCard spot={makeSpot()} onClick={() => {}} />)
    // 没有 .sc-foot 部分
    expect(screen.queryByText('¥')).not.toBeInTheDocument()
  })

  test('点击时调用 onClick', () => {
    const mockOnClick = jest.fn()  // jest.fn() 创建一个模拟函数，记录调用情况
    render(<SpotCard spot={makeSpot()} onClick={mockOnClick} />)

    fireEvent.click(screen.getByText('玄武湖'))

    expect(mockOnClick).toHaveBeenCalledTimes(1)  // 被调用了1次
  })

  test('住宿类型显示住宿图标', () => {
    render(<SpotCard spot={makeSpot({ type: 'hotel' })} onClick={() => {}} />)
    expect(screen.getByText('🏨')).toBeInTheDocument()
  })

  test('交通类型显示交通图标', () => {
    render(<SpotCard spot={makeSpot({ type: 'transport' })} onClick={() => {}} />)
    expect(screen.getByText('🚆')).toBeInTheDocument()
  })
})
```

## 测试带状态的组件

```tsx
// src/components/SpotSearch/SpotSearch.test.tsx
import { render, screen, fireEvent, waitFor } from '@testing-library/react'
import userEvent from '@testing-library/user-event'
import SpotSearch from './index'

// Mock 云函数调用
jest.mock('../../utils/cloud', () => ({
  cloud: {
    searchPoi: jest.fn(),
  },
}))

import { cloud } from '../../utils/cloud'

describe('SpotSearch', () => {
  const mockOnClose = jest.fn()
  const mockOnSelect = jest.fn()

  beforeEach(() => {
    jest.clearAllMocks()  // 每个测试前清空 mock 记录
  })

  test('open=false 时不渲染', () => {
    render(
      <SpotSearch open={false} onClose={mockOnClose} onSelect={mockOnSelect} />
    )
    expect(screen.queryByText('添加地点')).not.toBeInTheDocument()
  })

  test('open=true 时渲染搜索框', () => {
    render(
      <SpotSearch open={true} onClose={mockOnClose} onSelect={mockOnSelect} />
    )
    expect(screen.getByText('添加地点')).toBeInTheDocument()
    expect(screen.getByPlaceholderText('搜索地点,例:德基广场 / 玄武湖')).toBeInTheDocument()
  })

  test('点击关闭按钮调用 onClose', () => {
    render(
      <SpotSearch open={true} onClose={mockOnClose} onSelect={mockOnSelect} />
    )
    fireEvent.click(screen.getByText('×'))
    expect(mockOnClose).toHaveBeenCalledTimes(1)
  })

  test('输入关键词后显示搜索结果', async () => {
    // 模拟云函数返回搜索结果
    ;(cloud.searchPoi as jest.Mock).mockResolvedValue({
      results: [
        { name: '玄武湖', address: '南京市玄武区', city: '南京', adcode: '320102', lat: 32.07, lng: 118.80 },
        { name: '玄武门', address: '南京市玄武区', city: '南京', adcode: '320102', lat: 32.06, lng: 118.79 },
      ]
    })

    render(
      <SpotSearch open={true} onClose={mockOnClose} onSelect={mockOnSelect} />
    )

    const input = screen.getByPlaceholderText('搜索地点,例:德基广场 / 玄武湖')
    await userEvent.type(input, '玄武湖')

    // 等待防抖 + 异步请求完成
    await waitFor(() => {
      expect(screen.getByText('玄武湖')).toBeInTheDocument()
    })

    expect(screen.getByText('玄武门')).toBeInTheDocument()
  })

  test('选择结果后调用 onSelect 并关闭', async () => {
    ;(cloud.searchPoi as jest.Mock).mockResolvedValue({
      results: [
        { name: '玄武湖', city: '南京', adcode: '320102', lat: 32.07, lng: 118.80 }
      ]
    })

    render(
      <SpotSearch open={true} onClose={mockOnClose} onSelect={mockOnSelect} />
    )

    await userEvent.type(
      screen.getByPlaceholderText('搜索地点,例:德基广场 / 玄武湖'),
      '玄武湖'
    )

    await waitFor(() => screen.getByText('玄武湖'))
    fireEvent.click(screen.getByText('玄武湖'))

    expect(mockOnSelect).toHaveBeenCalledWith(
      expect.objectContaining({ name: '玄武湖', city: '南京' })
    )
    expect(mockOnClose).toHaveBeenCalled()
  })
})
```

## Mock（模拟）

测试组件时，不能真的发云函数请求（太慢、有副作用）。用 `jest.mock` 替换真实实现：

```javascript
// 模拟整个模块
jest.mock('../../utils/cloud', () => ({
  cloud: {
    searchPoi: jest.fn().mockResolvedValue({ results: [] }),
    getWeather: jest.fn().mockResolvedValue({ weather: null }),
  },
}))

// 在测试里动态调整返回值
;(cloud.searchPoi as jest.Mock).mockResolvedValue({
  results: [{ name: '玄武湖', ... }]
})

// 模拟失败情况
;(cloud.searchPoi as jest.Mock).mockRejectedValue(new Error('网络错误'))
```

### 模拟 Taro API

Taro 的 API 在测试环境里不存在，需要 mock：

```javascript
// jest.setup.ts（在 jest.config.js 里配置为 setupFilesAfterFramework）
jest.mock('@tarojs/taro', () => ({
  showToast: jest.fn(),
  showModal: jest.fn().mockResolvedValue({ confirm: true }),
  navigateTo: jest.fn(),
  useRouter: () => ({ params: { id: 'test-trip-id' } }),
}))
```

## 测试带 Context 的组件

行册的很多组件依赖 `useTripStore()`，测试时需要提供 Context：

```tsx
// 测试辅助：包裹 Context
function renderWithStore(ui: React.ReactElement, trip = makeTrip()) {
  const mockDispatch = jest.fn()
  const mockStore = { state: { trip, loading: false, error: null }, dispatch: mockDispatch }

  return {
    ...render(
      <TripStoreContext.Provider value={mockStore}>
        {ui}
      </TripStoreContext.Provider>
    ),
    mockDispatch,
  }
}

// 在测试里使用
test('点击添加地点时 dispatch ADD_SPOT', async () => {
  const { mockDispatch } = renderWithStore(<ItineraryView />)

  // 模拟搜索和选择地点
  fireEvent.click(screen.getByText('+ 添加地点'))
  // ...

  expect(mockDispatch).toHaveBeenCalledWith(
    expect.objectContaining({ type: 'ADD_SPOT' })
  )
})
```

## 快照测试（Snapshot Testing）

快照测试记录组件渲染的「快照」，之后每次运行都对比是否有变化：

```tsx
import { render } from '@testing-library/react'

test('SpotCard 渲染快照', () => {
  const { container } = render(
    <SpotCard
      spot={{ id: '1', type: 'spot', name: '玄武湖', price: 0 }}
      onClick={() => {}}
    />
  )
  expect(container).toMatchSnapshot()
})
```

第一次运行会生成快照文件（`__snapshots__/SpotCard.test.tsx.snap`）。之后组件变化时，测试会失败并提示你确认这个改变是否是预期的。

```bash
# 更新快照（当你确认 UI 改动是有意的）
npx jest --updateSnapshot
```

快照测试的缺点：太容易误报（一点小改动就失败），不要过度使用。

## 行册测试优先级建议

根据价值和难度，按这个顺序来写测试：

| 优先级 | 目标文件 | 理由 |
|--------|---------|------|
| ⭐⭐⭐ 高 | `utils/format.ts` | 纯函数，10 分钟写完 |
| ⭐⭐⭐ 高 | `utils/trip-helpers.ts` | 纯函数，核心业务逻辑 |
| ⭐⭐⭐ 高 | `views/MapView/helpers.ts` | 纯函数，有 ID 编码逻辑 |
| ⭐⭐⭐ 高 | `store/trip-store.tsx`（Reducer）| 核心状态逻辑 |
| ⭐⭐ 中 | `views/ItineraryView/SpotCard.tsx` | 简单展示组件 |
| ⭐⭐ 中 | `components/SpotSearch/` | 有搜索逻辑 |
| ⭐ 低 | 依赖 Taro Map 的组件 | 难以 mock |

## 完整测试命令

```bash
# 运行所有测试
npx jest

# 只运行某个文件
npx jest format.test.ts

# 按名字过滤测试
npx jest -t 'fmtCurrency'

# 监听变化自动重跑
npx jest --watch

# 生成覆盖率报告（HTML 格式）
npx jest --coverage --coverageReporters html
# 然后打开 coverage/lcov-report/index.html 查看
```

## 小结

| 场景 | 工具 |
|------|------|
| 渲染组件 | `render()` |
| 查找元素 | `screen.getBy*()`, `screen.queryBy*()` |
| 触发事件 | `fireEvent.click()`, `userEvent.type()` |
| 等待异步 | `waitFor()`, `findBy*()` |
| 模拟函数 | `jest.fn()` |
| 模拟模块 | `jest.mock()` |
| 断言 | `expect().toBe()`, `.toBeInTheDocument()` |

**测试不是负担，是让你敢于修改代码的底气。**  
从最简单的工具函数开始，逐步建立测试习惯。
