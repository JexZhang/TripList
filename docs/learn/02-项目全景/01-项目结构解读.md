# 第二章 · 项目结构解读

打开 `TripList/` 文件夹，你会看到这些东西。本章逐一解释每个文件夹和文件的作用。

---

## 顶层目录

```
TripList/
├── src/                 ← 🟢 你写代码的地方（最重要）
├── cloudfunctions/      ← 🟢 后端云函数（服务器逻辑）
├── dist/                ← 🔵 编译输出（自动生成，不要手改）
├── docs/                ← 📖 文档（包括这份学习手册）
├── node_modules/        ← 📦 依赖包（不要手改）
├── config/              ← ⚙️ Taro 构建配置
├── package.json         ← 📋 项目配置（依赖、命令）
├── tsconfig.json        ← ⚙️ TypeScript 配置
└── project.config.json  ← ⚙️ 微信小程序配置
```

## pages/ vs views/ vs components/ 的区别

这是初学者最容易混淆的部分，用一张图说清楚：

```
┌─────────────────────────────────────────┐
│  页面 (page/trip/index.tsx)              │
│  ┌────────────────────────────────────┐ │
│  │  顶部头部（标题、日期、协作者栏）      │ │
│  │  Tab 导航栏（攻略|开销|清单|地图）    │ │
│  │  ┌──────────────────────────────┐ │ │
│  │  │  视图 (view/ItineraryView)    │ │ │
│  │  │  ┌───────────┐ ┌──────────┐ │ │ │
│  │  │  │ 组件      │ │  组件    │ │ │ │
│  │  │  │ SpotCard  │ │SpotSearch│ │ │ │
│  │  │  └───────────┘ └──────────┘ │ │ │
│  │  └──────────────────────────────┘ │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

- **pages**：完整的一屏，微信的「路由」可以直接跳转到这里
- **views**：页面内 Tab 切换的不同「视图」，不能独立访问
- **components**：可以在任何地方复用的小块（弹窗、卡片等）

## 每个 pages/ 子文件夹的内容

每个页面通常有三个文件：

```
pages/trip/
├── index.tsx        ← 核心：JSX 结构和逻辑
├── index.config.ts  ← 页面配置（标题栏颜色等）
└── index.scss       ← 这个页面专属的样式
```

比如 `trip/index.config.ts`：
```typescript
export default definePageConfig({
  navigationBarTitleText: '攻略',          // 顶部标题
  navigationBarBackgroundColor: '#f7f1e3', // 标题栏背景色
  navigationBarTextStyle: 'black',         // 标题文字颜色
})
```

## app.config.ts — 路由表

这是小程序的「目录」，告诉微信有哪些页面：

```typescript
// src/app.config.ts
export default defineAppConfig({
  pages: [
    'pages/home/index',     // 首页（第一个是启动页）
    'pages/new-trip/index', // 新建攻略
    'pages/trip/index',     // 攻略详情
    'pages/share/index',    // 分享接收
  ],
  window: {
    navigationBarBackgroundColor: '#f7f1e3',
    navigationBarTitleText: '行册',
    navigationBarTextStyle: 'black'
  }
})
```

## 一个文件的完整解剖

以 `src/utils/format.ts` 为例（最简单的工具文件）：

```typescript
import dayjs from 'dayjs'  // 导入第三方库（处理日期）

// 格式化日期
export function fmtDate(date: string, pattern = 'YYYY-MM-DD'): string {
  return dayjs(date).format(pattern)
}

// 格式化短日期：05.29
export function fmtDateShort(date: string): string {
  return dayjs(date).format('MM.DD')
}

// 格式化金额：¥1,234
export function fmtCurrency(n: number, currency = '¥'): string {
  return currency + n.toLocaleString()
}
```

- `import` — 从其他文件引入代码
- `export` — 让这个函数可以被其他文件使用
- `function 名字(参数: 类型): 返回类型` — TypeScript 的函数写法

## 命名规范

行册遵循这些命名习惯，看代码时一眼认出：

| 规则 | 例子 | 用于 |
|------|------|------|
| `PascalCase` | `TripCard`, `EditSpotSheet` | 组件名、类型名 |
| `camelCase` | `listMyTrips`, `keyboardHeight` | 函数名、变量名 |
| `kebab-case` | `trip-card`, `ss-input` | CSS 类名 |
| `UPPER_SNAKE` | `PACKING_TEMPLATES` | 常量 |

## 文件引入关系

```
app.tsx
  └─ pages/home/index.tsx
  └─ pages/trip/index.tsx
       └─ views/ItineraryView/index.tsx
            └─ components/SpotSearch/index.tsx
            └─ components/EditSpotSheet/index.tsx
       └─ views/MapView/index.tsx
            └─ MapView/SpotMapSheet.tsx
       └─ store/trip-store.tsx
            └─ utils/db.ts
            └─ utils/cloud.ts
```

这种树状引入关系，就是「组件树」，React 的核心概念。

## 小结

- `src/pages/` — 页面，可以导航到这里
- `src/views/` — Tab 视图，存在于某个页面内
- `src/components/` — 复用的 UI 块
- `src/store/` — 全局状态
- `src/utils/` — 工具函数
- `cloudfunctions/` — 后端逻辑
