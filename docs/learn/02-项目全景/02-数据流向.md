# 第二章 · 数据流向

「数据从哪来，到哪去，怎么变化」——理解这条线，就理解了整个应用。

## 一、整体架构

```
┌─────────────────────────────────────────────────────┐
│                    你的手机（前端）                     │
│  ┌─────────────┐    ┌──────────┐    ┌────────────┐  │
│  │  React 组件  │◄──│  Store   │◄──│  utils/db  │  │
│  │  (界面渲染)  │──►│  (状态)  │──►│  (数据库层) │  │
│  └─────────────┘    └──────────┘    └────────────┘  │
│                                           │ ↑        │
└───────────────────────────────────────────┼─┼────────┘
                                            │ │ 网络请求
┌───────────────────────────────────────────┼─┼────────┐
│                    微信云端（后端）          ↓ │        │
│  ┌──────────────────┐    ┌──────────────────────┐    │
│  │   云函数           │    │     云数据库           │    │
│  │ (业务逻辑/权限校验) │◄──►│  (存储所有攻略数据)   │    │
│  └──────────────────┘    └──────────────────────┘    │
└─────────────────────────────────────────────────────┘
```

## 二、场景追踪：用户添加一个地点

让我们追踪一个完整操作：用户在攻略里点「添加地点」，搜索「德基广场」，点击确认。

### Step 1：用户点击按钮

```tsx
// src/views/ItineraryView/index.tsx
<View className='itin-add-spot' onClick={() => setSearchOpen(true)}>
  + 添加地点
</View>
```

用户点击 → `setSearchOpen(true)` → `searchOpen` 状态变为 `true`

### Step 2：搜索弹窗出现

因为 `searchOpen` 变成 `true`，React 重新渲染：

```tsx
{/* open=true 时才渲染这个组件 */}
<SpotSearch
  open={searchOpen}
  onClose={() => setSearchOpen(false)}
  onSelect={handleAddSpot}
/>
```

### Step 3：用户输入关键词

```tsx
// src/components/SpotSearch/index.tsx
<Input
  value={keyword}
  onInput={e => setKeyword(e.detail.value)}  // 输入框每次变化都更新 keyword
/>
```

用户输入「德基广场」→ `keyword` 状态变成 `'德基广场'`

### Step 4：触发搜索（防抖）

```tsx
// 300ms 防抖：等用户停止输入再发请求
useEffect(() => {
  const t = setTimeout(() => setDebouncedKw(keyword), 300)
  return () => clearTimeout(t)
}, [keyword])

// 发网络请求
useEffect(() => {
  cloud.searchPoi({ keyword: debouncedKw })  // ← 调云函数
    .then(r => setResults(r.results))
}, [debouncedKw])
```

### Step 5：云函数处理请求

```javascript
// cloudfunctions/amap-poi-search/index.js
exports.main = async (event) => {
  const { keyword, city } = event          // 接收前端传来的参数
  const response = await axios.get(        // 调高德地图 API
    `https://restapi.amap.com/v5/place/text?keywords=${keyword}&key=${AMAP_KEY}`
  )
  return { results: response.data.pois }   // 返回搜索结果
}
```

### Step 6：结果显示

```tsx
// 展示搜索结果
{results.map(r => (
  <View key={r.adcode} onClick={() => useResult(r)}>
    <Text>{r.name}</Text>
    <Text>{r.address}</Text>
  </View>
))}
```

### Step 7：用户选择，地点加入行程

```tsx
// src/views/ItineraryView/index.tsx
const handleAddSpot = (info) => {
  const spot = {
    id: uid(),       // 生成唯一 ID
    type: 'spot',
    name: info.name,
    lat: info.lat,
    lng: info.lng,
  }
  dispatch({ type: 'ADD_SPOT', dayId: activeDay.id, spot })
  //  ↑ 发送 Action 给 Reducer，更新全局状态
}
```

### Step 8：Store 更新状态

```tsx
// src/store/trip-store.tsx
case 'ADD_SPOT': {
  return {
    ...state,
    trip: {
      ...state.trip,
      days: state.trip.days.map(day =>
        day.id === action.dayId
          ? { ...day, spots: [...day.spots, action.spot] }  // 把新地点加进去
          : day
      )
    }
  }
}
```

### Step 9：自动保存到云数据库

```tsx
// trip-store 里有个 useEffect 监听 trip 变化，500ms 后自动保存
useEffect(() => {
  const timer = setTimeout(() => {
    updateTrip(trip._id, { days: trip.days }, openid)  // 保存到云端
  }, 500)
  return () => clearTimeout(timer)
}, [trip])
```

### Step 10：界面更新

Store 状态更新 → React 重新渲染 → 新地点出现在界面上

## 三、数据流总结图

```
用户操作（点击/输入）
        ↓
   组件内的事件处理函数
        ↓
   dispatch(action) 或 setState
        ↓
   ┌────────────────────────┐
   │   Store (Reducer)      │
   │   处理 action，更新数据  │
   └────────────────────────┘
        ↓              ↓
  界面重新渲染       自动保存（防抖）
                        ↓
                   云函数 update-trip
                        ↓
                     云数据库
```

## 四、数据「从哪来」

行册有三种数据来源：

### 1. 云数据库（主要数据源）

```
云数据库 → list-my-trips 云函数 → db.ts 的 listMyTrips() → Home 页面

```

## 五、「单向数据流」是什么

React 的核心原则：**数据只往一个方向流动**。

```
数据（State）→ 渲染（View）→ 操作（Action）→ 更新数据（State）→ ...
```

不允许：组件 B 直接修改组件 A 的数据。  
正确做法：组件 B 调用 A 传来的回调函数，A 自己更新自己的数据。

```tsx
// ❌ 错误：子组件直接修改父组件的数据
function Child({ parentState }) {
  parentState.value = 'new'  // 不能这样！
}

// ✅ 正确：通过回调函数通知父组件
function Child({ onValueChange }) {
  onValueChange('new')  // 告诉父组件，让父组件自己改
}
```

## 六、实时同步：多人协作时

当你和朋友同时编辑同一个攻略，数据怎么同步？

```tsx
// src/store/trip-store.tsx
// 监听数据库变化
const watcher = db.collection('trips').doc(tripId).watch({
  onChange(snapshot) {
    const updated = snapshot.docs[0]
    if (isPending) return  // 忽略自己刚刚做的改动
    dispatch({ type: 'SET_TRIP', trip: updated })  // 收到别人的更新
  }
})
```

流程：
```
你朋友修改地点
     ↓
朋友的手机 → update-trip 云函数 → 云数据库保存
                                        ↓
                                  数据库 watch 触发通知
                                        ↓
                                   你的手机收到更新
                                        ↓
                                   dispatch SET_TRIP
                                        ↓
                                   你的界面自动更新
```

## 七、防抖（Debounce）：为什么不立刻保存

如果每次用户输入一个字就保存一次，会产生大量无意义的网络请求。  
防抖的思路：「等用户停下来再保存」

```
用户快速输入 "南京市玄武湖"：
  s → 计时 500ms
  sn → 重置计时 500ms
  sna → 重置计时 500ms
  ...
  南京市玄武湖 → 重置计时 500ms
                （停止输入）
                500ms 后 → 发送一次保存请求 ✅
```

比每个字保存一次，减少了 95% 以上的网络请求。

## 小结

数据流的关键链路：

1. **读数据**：云数据库 → 云函数 → `utils/db.ts` → Store → 组件渲染
2. **写数据**：用户操作 → dispatch → Store 更新 → 防抖 → `utils/db.ts` → 云函数 → 云数据库
3. **实时同步**：数据库 watch → dispatch → 组件自动更新
4. **第三方数据**：高德 API → 云函数代理 → 组件显示
