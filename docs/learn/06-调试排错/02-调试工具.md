## Console 面板

最常用的调试面板，查看 `console.log` 输出和错误信息。

**如何打开**：底部「调试器」→「Console」

### 过滤日志

Console 左上角有过滤器：
- `Errors` — 只看错误（红色）
- `Warnings` — 只看警告（黄色）
- 搜索框 — 搜索关键词

### 在 Console 里直接执行代码

可以在 Console 里输入 JavaScript 代码直接执行，方便测试：

```javascript
// 测试工具函数
// 先在 Console 里输入：
fmtCurrency(1234)    // → '¥1,234'

// 查看当前页面的 app 实例
getApp()

// 查看当前页面数据（原生小程序）
getCurrentPages()[0].data
```

## Sources 面板

可以设置断点、单步调试。

**断点调试**比 `console.log` 更强大——它能暂停代码执行，让你查看当时所有变量的值。

### 设置断点

1. 在 Sources 面板左边找到你的文件
2. 点击行号，出现蓝色标记 = 断点

### 调试步骤

代码运行到断点处暂停后：
- **▶ 继续** — 运行到下一个断点
- **↓ 单步（Step Over）** — 执行当前行，跳到下一行
- **↘ 进入（Step In）** — 进入函数内部
- **↗ 跳出（Step Out）** — 跳出当前函数

右侧面板可以查看：
- **Scope** — 当前作用域里所有变量的值
- **Call Stack** — 函数调用链

### Taro 代码定位

Taro 编译后的代码难以阅读，但有 Source Map 支持：

开发模式（`npm run dev:weapp`）会生成 `.map` 文件，调试器能显示原始 TypeScript 代码而不是编译后的代码。

## Network 面板

查看所有网络请求，包括云函数调用。

**重要**：微信开发者工具里，点击某个请求可以看到：
- 请求参数（发了什么）
- 响应数据（收到什么）
- 耗时（多长时间）

调试云函数时：
1. 触发某个操作（比如搜索地点）
2. Network 里找到 `callFunction`
3. 查看 Request Payload（发送的参数）
4. 查看 Response（返回的数据）

如果云函数返回错误，这里能看到具体的错误信息。

## Storage 面板

查看小程序的本地存储（`wx.setStorageSync` / `wx.getStorageSync`）。

行迹暂时没有明显的 localStorage 使用，但云开发缓存有时会存在这里。

## AppData 面板

在原生小程序开发中，AppData 显示当前 `Page.data` 的内容。  
由于行迹用了 Taro + React，这里通常没有有用信息，主要看 Console。

## 云开发控制台

点击顶部「云开发」按钮进入云开发控制台。

### 查看云函数日志

1. 「云函数」→ 选择函数（如 `update-trip`）→ 「日志」
2. 可以看到：
   - 调用时间
   - 函数输出（`console.log`）
   - 运行时间
   - 错误信息

**注意**：云函数日志有延迟（约 1-2 分钟才显示），不是实时的。

### 查看数据库数据

1. 「数据库」→ 选择集合（如 `trips`）
2. 可以浏览、搜索文档
3. 也可以手动编辑数据（调试时有用）

比如：怀疑某个攻略数据有问题，直接去数据库找到那条记录看原始数据。

## VS Code 调试

除了微信开发者工具，可以在 VS Code 里调试 TypeScript 代码（主要用于调试工具函数）：

```json
// .vscode/launch.json
{
  "configurations": [
    {
      "type": "node",
      "request": "launch",
      "name": "调试工具函数",
      "program": "${workspaceFolder}/src/utils/format.ts",
      "runtimeArgs": ["-r", "ts-node/register"],
    }
  ]
}
```

## 真机调试

有时候模拟器里没问题，但真机上有 Bug，需要真机调试。

### 方法 1：预览码

1. 开发者工具点「预览」→ 用手机扫码
2. 手机上运行时，开发者工具的 Console 面板会显示手机上的日志

### 方法 2：真机调试

1. 开发者工具点「真机调试」→ 用手机扫码
2. 手机上的操作可以在电脑上看到 Console 输出、Sources 断点等

## TypeScript 类型提示辅助调试

TypeScript 编译期间的错误提示能帮你在运行前发现问题：

```tsx
// 假设你不小心写了：
const totalPrice = spots.reduce((sum, spot) => sum + spot.price, '')
//                                                              ↑ 初始值给了字符串，但应该是数字

// TypeScript 会提示：
// Argument of type 'string' is not assignable to parameter of type 'number'
```

在 VS Code 里，TypeScript 错误会在代码下方显示红色波浪线，鼠标悬停可以看到提示。

## ESLint 代码检查

行迹配置了 ESLint，运行时会检查代码规范问题：

```bash
npm run lint
# 输出例子：
# src/views/ItineraryView/index.tsx
#   45:5  warning  'activeDayId' is defined but never used  @typescript-eslint/no-unused-vars
```

常见的 ESLint 警告：
- `is defined but never used` — 定义了变量但没用，可能是遗漏了使用
- `React Hook useEffect has a missing dependency` — useEffect 依赖数组不完整，可能有 Bug

## 常用调试命令

```bash
# 运行开发模式（有热更新）
npm run dev:weapp

# 检查 TypeScript 类型错误
npx tsc --noEmit

# 检查代码规范
npm run lint

# 检查样式规范
npm run lint:style
```
