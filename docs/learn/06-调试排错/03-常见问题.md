# 第六章 · 常见问题

收集行册开发中遇到过的典型问题，以及解决方法。

---

## 界面问题

### Q1：弹窗出现后键盘把内容顶没了

**现象**：点输入框，键盘弹出，但搜索结果和按钮被键盘遮住了，看不见。

**原因**：
- 弹窗是 `position: fixed`，固定在屏幕底部
- 微信默认的 `adjust-position=true` 会推整个页面上移
- 但 fixed 元素不跟着页面移动，所以内容被键盘覆盖

**解决**：
```tsx
<Input
  adjustPosition={false}           // 关闭默认上推
  // @ts-ignore
  onKeyboardHeightChange={e => setKeyboardHeight(e.detail.height)}
/>

// 弹窗容器
<View
  className='spot-search-mask'
  style={{ paddingBottom: `${keyboardHeight}px`, transition: 'padding-bottom 0.25s ease' }}
>
```

---

### Q2：页面可以横向滑出去，出现灰色区域

**现象**：在攻略或地图页面，向左/右滑动时，整个页面偏移，右边出现灰色空白。

**原因**：微信小程序的页面默认允许横向滚动（CSS 没有禁止）。

**解决**：
```scss
// app.scss
page {
  overflow-x: hidden;
}

// pages/trip/index.scss
.trip {
  overflow-x: hidden;
}
```

---

### Q3：ScrollView 不滚动

**现象**：设置了 `scrollY`，但内容超出后没有出现滚动。

**原因**：ScrollView 必须有明确的高度才能滚动。

**解决**：
```scss
// 方法1：固定高度
.ss-results {
  height: 300rpx;  /* 固定高度 */
  overflow-y: scroll;
}

// 方法2：flex 布局（行册用的）
.spot-search-sheet {
  height: 50vh;
  display: flex;
  flex-direction: column;
}
.ss-results {
  flex: 1;       /* 占满剩余空间 */
  height: 0;     /* 重要！没有这行 flex 子元素不会限高 */
  min-height: 0;
}
```

---

### Q4：动画闪烁或抖动

**现象**：Sheet 弹出时有短暂的闪烁，或位置不对。

**原因**：动画和渲染时机的竞争。

**解决**：用 CSS `animation-fill-mode: both`，确保动画开始前就应用初始状态：
```scss
.edit-spot-sheet {
  animation: sheet-up 0.4s var(--ease-spring) both;  /* both = 保持起始和结束状态 */
}
```

## 数据问题

### Q5：修改数据后界面没有更新

**现象**：dispatch 了 action，但界面没变化。

**可能原因 1**：直接修改了 state（违反不可变原则）
```tsx
// ❌ 错误
state.trip.name = '新名字'  // 直接修改，React 检测不到变化

// ✅ 正确
return { ...state, trip: { ...state.trip, name: '新名字' } }
```

**可能原因 2**：Reducer 没有匹配的 case
```tsx
switch (action.type) {
  case 'UPDATE_SPOT':
    // ...
  // 忘了写 default！导致其他 action 返回 undefined
  default:
    return state  // ← 这行很重要
}
```

**可能原因 3**：忘记 dispatch
```tsx
// 调用了函数但忘了 dispatch
const handleAddSpot = (info) => {
  const spot = { ... }
  // 忘了 dispatch！
  // dispatch({ type: 'ADD_SPOT', dayId: activeDay.id, spot })
}
```

---

### Q6：数据保存了，但刷新后丢失

**现象**：添加地点后界面上有，但退出重进攻略，地点消失了。

**排查**：
1. 打开开发者工具 Network 面板
2. 触发保存操作
3. 找到 `update-trip` 云函数的调用
4. 查看返回值是否是 `{ ok: true }`

**常见原因**：
- 防抖 500ms 没等完就退出了页面，保存没触发
- 云函数报错但前端没有处理
- 数据库权限问题（协作者权限被拒绝）

---

### Q7：多人协作时数据覆盖

**现象**：你和朋友同时编辑，你的修改被覆盖了。

**原因**：行册用「最后写入者获胜」策略，这是已知的设计局限。

**缓解方法**：协作时分工编辑不同的天，减少冲突概率。

## 云函数问题

### Q8：云函数调用失败

**现象**：操作后没有反应，或者出现「操作失败」提示。

**排查步骤**：
1. 打开微信开发者工具的 Console，找红色错误
2. 查看云开发控制台的云函数日志
3. 在云函数里加更多 `console.log` 找到出错位置

**常见原因**：
```javascript
// ❌ 错误：忘记 await
const trip = db.collection('trips').doc(tripId).get()  // 返回的是 Promise，不是数据！
console.log(trip.data)  // undefined

// ✅ 正确
const trip = await db.collection('trips').doc(tripId).get()
console.log(trip.data)
```

---

### Q9：云函数权限被拒绝

**现象**：`update-trip` 云函数返回 `{ error: '没有权限' }`。

**排查**：
1. 确认当前用户的 openid 和攻略的 `_openid` 或 `collaboratorOpenids`
2. 在云开发控制台手动查询数据库，看 `collaboratorOpenids` 里是否包含当前用户

---

### Q10：高德地图搜索没结果

**现象**：搜索地点，转圈很久然后没有结果。

**可能原因 1**：AMAP_KEY 没有配置
```
云开发控制台 → 云函数 → amap-poi-search → 配置 → 环境变量
确认 AMAP_KEY 已设置
```

**可能原因 2**：高德 API 配额用完

**可能原因 3**：网络问题（云函数超时）

在 `amap-poi-search` 云函数日志里可以看到具体错误。

## TypeScript 问题

### Q11：TypeScript 报错但不影响运行

```
Property 'onKeyboardHeightChange' does not exist on type 'InputProps'
```

Taro 的类型定义有时落后于实际 API，这种情况用 `@ts-ignore` 跳过：

```tsx
<Input
  // @ts-ignore - Taro 类型定义暂不包含此属性，但运行时有效
  onKeyboardHeightChange={e => setKeyboardHeight(e.detail.height)}
/>
```

---

### Q12：`Cannot find module` 错误

```
Cannot find module '../../utils/cloud' or its corresponding type declarations
```

**可能原因**：
1. 文件路径写错了（大小写敏感！）
2. 文件还没创建
3. `tsconfig.json` 的 `paths` 配置问题

## 性能问题

### Q13：地图渲染后视野跳动

**现象**：地图加载后，视野突然变化，用户体验很差。

**原因**：Map 组件挂载后，`includePoints` 才调用，导致视野跳变。

**已有的解决**：行册用 `useMemo` 缓存 markers 和 polyline，用 `SheetContainer` 隔离状态，减少不必要的重渲染。

---

### Q14：列表渲染很慢

**现象**：攻略里有很多地点时，界面卡顿。

**解决思路**：
1. 检查是否缺少 `key` 属性（缺 key 会导致全量重渲染）
2. 用 `useMemo` 缓存昂贵的计算
3. 考虑虚拟列表（当地点超过 100 个时）

## 小结

| 问题类型 | 最常见原因 | 首先检查的地方 |
|---------|----------|------------|
| 界面不更新 | 直接修改 state | Reducer 的返回值 |
| 数据丢失 | 保存时机问题 | Network 面板 |
| 弹窗样式问题 | flex 高度 / fixed 定位 | 检查 CSS |
| 云函数失败 | 权限/配置/await 遗漏 | 云函数日志 |
| TypeScript 报错 | 类型不匹配/路径错误 | VS Code 提示 |
