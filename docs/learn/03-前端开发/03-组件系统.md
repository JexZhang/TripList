# 第三章 · 组件系统

## 什么是「好的组件设计」

好的组件像乐高积木：
- 每块积木只做一件事
- 积木可以自由组合
- 换一块积木，其他积木不受影响

---

## 行迹的组件分层

```
页面层 (pages/)         ← 一整屏，负责「编排」
    ↓ 使用
视图层 (views/)         ← Tab 内容，负责「区域」
    ↓ 使用
组件层 (components/)    ← 可复用积木，负责「具体功能」
```

## 单文件组件结构

每个组件通常由两个文件组成：

```
EditSpotSheet/
├── index.tsx    ← 逻辑 + JSX
└── index.scss   ← 专属样式
```

`index.tsx` 的典型结构：

```tsx
// 1. 导入
import { useState, useEffect } from 'react'
import { View, Text, Input } from '@tarojs/components'
import type { Spot } from '../../types/trip'
import './index.scss'  // 导入样式

// 2. 定义 Props 类型
interface Props {
  open: boolean
  spot: Spot | null
  onClose: () => void
  onSave: (patch: Partial<Spot>) => void
}

// 3. 组件函数
export default function EditSpotSheet({ open, spot, onClose, onSave }: Props) {
  // 4. 内部状态
  const [draft, setDraft] = useState<Partial<Spot>>({})

  // 5. 副作用
  useEffect(() => {
    if (open && spot) setDraft({ ...spot })
  }, [open, spot])

  // 6. 早返回（条件不满足时不渲染）
  if (!open || !spot) return null

  // 7. 事件处理函数
  const save = () => {
    onSave(draft)
    onClose()
  }

  // 8. JSX 渲染
  return (
    <View className='edit-spot-mask' onClick={onClose}>
      <View className='edit-spot-sheet' onClick={e => e.stopPropagation()}>
        {/* 内容 */}
      </View>
    </View>
  )
}
```

## 常见组件模式

### 模式一：弹窗/Sheet

行迹有很多底部弹出的「Sheet」，都遵循相同模式：

```tsx
// 结构：遮罩层 + 内容层
<View className='edit-spot-mask' onClick={onClose}>     {/* 遮罩：点击关闭 */}
  <View className='edit-spot-sheet'
    onClick={e => e.stopPropagation()}  {/* 阻止冒泡，点内容不关闭 */}
  >
    {/* 实际内容 */}
  </View>
</View>
```

```scss
/* 遮罩：全屏半透明，从底部对齐内容 */
.edit-spot-mask {
  position: fixed; inset: 0;          /* 覆盖整个屏幕 */
  background: rgba(43, 26, 16, 0.45); /* 半透明棕色 */
  display: flex; align-items: flex-end; /* 内容从底部显示 */
  z-index: 90;
  animation: fade-in 0.22s var(--ease-out) both;
}

/* 内容：白色卡片，从底部滑入 */
.edit-spot-sheet {
  width: 100%;
  height: 50vh;        /* 占屏幕一半高 */
  background: var(--bg);
  border-radius: 40rpx 40rpx 0 0;  /* 只有上方圆角 */
  animation: sheet-up 0.4s var(--ease-spring) both;  /* 弹入动画 */
}
```

### 模式二：可控组件（受控输入框）

```tsx
// 受控：值由父组件控制
const [keyword, setKeyword] = useState('')

<Input
  value={keyword}                          // 显示当前值
  onInput={e => setKeyword(e.detail.value)} // 每次输入都更新状态
/>
```

### 模式三：列表 + 空状态

```tsx
// 有数据时显示列表，没数据时显示提示
{spots.length > 0 ? (
  spots.map(s => <SpotCard key={s.id} spot={s} />)
) : (
  <View className='itin-empty'>这一天还没有地点</View>
)}

// 或者更简洁：
{spots.map(s => <SpotCard key={s.id} spot={s} />)}
{spots.length === 0 && <View className='itin-empty'>这一天还没有地点</View>}
```

### 模式四：Loading 状态

```tsx
const [loading, setLoading] = useState(false)

const loadData = async () => {
  setLoading(true)
  try {
    const data = await fetchData()
    setData(data)
  } finally {
    setLoading(false)  // 无论成功失败都要取消 loading
  }
}

// 渲染
{loading ? <Text>加载中...</Text> : <DataView data={data} />}
```

## 组件通信

### 父 → 子：通过 Props

```tsx
// 父组件传数据给子组件
function Parent() {
  const [spots, setSpots] = useState([...])

  return <SpotList spots={spots} />  // 传数据
}

function SpotList({ spots }) {
  return spots.map(s => <SpotCard spot={s} />)
}
```

### 子 → 父：通过回调 Props

```tsx
// 子组件通过调用 props 里的函数，把数据传回父组件
function SpotSearch({ onSelect }) {
  const handleSelect = (result) => {
    onSelect(result)  // 调用父组件传来的函数
  }

  return <ResultItem onClick={() => handleSelect(r)} />
}

function Parent() {
  const handleAddSpot = (info) => {
    // 收到子组件传来的数据
    addSpotToTrip(info)
  }

  return <SpotSearch onSelect={handleAddSpot} />
}
```

### 跨层级：通过 Context（Store）

如果数据需要在很多层级之间共享（比如攻略数据），用 Context：

```tsx
// 任何层级都可以直接取数据，不需要一层层传
function DeepNestedComponent() {
  const { state, dispatch } = useTripStore()  // 直接取！
  return <Text>{state.trip.name}</Text>
}
```

## 组件拆分的判断标准

什么时候应该把一块 JSX 拆成独立组件？

✅ **应该拆分**：
- 相同的 JSX 在多处使用（复用性）
- 一个组件超过 150 行（可读性）
- 有独立的内部状态（内聚性）
- 变化频率不同（性能优化）

❌ **不用拆分**：
- 只用一次的简单 JSX
- 拆开后需要传很多 props（反而麻烦）

行迹里 `SheetContainer` 是个好例子：

```tsx
// src/views/MapView/index.tsx
// 把 SpotMapSheet 单独封装成 SheetContainer
// 原因：Map 和 Sheet 的状态独立，这样 Sheet 开/关时不会触发 Map 重新渲染
const SheetContainer = forwardRef(function SheetContainer(_props, ref) {
  const [picked, setPicked] = useState(null)
  useImperativeHandle(ref, () => ({
    show: (spot) => setPicked(spot),
  }))
  return <SpotMapSheet spot={picked} onClose={() => setPicked(null)} />
})
```

## `forwardRef` 和 `useImperativeHandle`：命令式调用

大多数情况下，父组件通过 props 控制子组件。但有时需要「命令式调用」（直接调用子组件的方法）：

```tsx
// 定义：子组件暴露方法
const SheetContainer = forwardRef(function SheetContainer(props, ref) {
  const [visible, setVisible] = useState(false)

  // 暴露给外部可以调用的方法
  useImperativeHandle(ref, () => ({
    show: () => setVisible(true),
    hide: () => setVisible(false),
  }))

  return <View>{visible && <Sheet />}</View>
})

// 使用：父组件直接调用
const sheetRef = useRef(null)

function Parent() {
  return (
    <>
      <Map onMarkerTap={() => sheetRef.current.show()} />
      <SheetContainer ref={sheetRef} />
    </>
  )
}
```

## 小结

- 组件 = 界面的可复用积木
- Props = 从外到内传数据
- 回调 Props = 从内向外通知
- Context = 跨层级共享数据
- 弹窗 = 遮罩 + 内容 + 点遮罩关闭
