# 第四章 · 云开发入门

## 传统后端 vs 云开发

**传统后端开发流程**：
1. 租服务器（阿里云/腾讯云）
2. 安装 Linux、Node.js、数据库
3. 写后端代码（Express/FastAPI 等）
4. 部署上线
5. 维护服务器（更新、扩容等）

**微信云开发**：
1. 在微信开发者工具里直接写「云函数」
2. 点一下「上传」
3. 完成！

微信帮你管好服务器、数据库、扩容等一切基础设施。

## 云开发的三个核心能力

### 1. 云数据库
NoSQL 数据库（类似 MongoDB），存储应用数据。  
行册把所有攻略、用户信息都存这里。

```javascript
// 在云函数里操作数据库
const db = cloud.database()

// 查询
const trips = await db.collection('trips')
  .where({ _openid: '{openid}' })
  .get()

// 插入
await db.collection('trips').add({ data: newTrip })

// 更新
await db.collection('trips').doc(tripId).update({ data: patch })

// 删除
await db.collection('trips').doc(tripId).remove()
```

### 2. 云函数
运行在服务器上的函数，处理业务逻辑。  
行册有 8 个云函数：`ensure-user`, `list-my-trips`, `update-trip` 等。

```javascript
// 云函数基本结构（Node.js）
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  // event：前端传来的参数
  // context：运行环境信息（包含调用者的 openid）
  const { OPENID } = cloud.getWXContext()  // 获取当前用户 ID

  // 在这里写业务逻辑
  const result = await doSomething(event.data)

  return { success: true, data: result }
}
```

### 3. 云存储
存图片、文件等二进制文件。  
行册暂时没有用到云存储（所有数据都是文字）。

## 前端怎么调用云函数

```tsx
// src/utils/cloud.ts
async function call<TIn, TOut>(name: string, data: TIn): Promise<TOut> {
  const res = await Taro.cloud.callFunction({ name, data })
  return res.result as TOut
}

// 使用
const result = await call('ensure-user', { nickname: '张三', avatarUrl: '' })
// 等价于调用了 cloudfunctions/ensure-user/index.js 的 main 函数
```

## 数据库的权限模型

云数据库的数据访问有严格权限控制：

| 规则 | 含义 |
|------|------|
| 仅创建者可读写 | 只有数据的 `_openid` 等于当前用户的 openid |
| 所有人可读，仅创建者可写 | 读没限制，写有限制 |
| 所有人可读写 | 完全开放（危险！） |

行册的 `trips` 集合默认权限是「仅创建者可读写」，但这导致协作者没法读写别人的攻略。  
解决方案：使用「云函数」来操作数据库——云函数有管理员权限，可以绕过客户端权限限制。

```javascript
// cloudfunctions/update-trip/index.js
// 云函数有管理员权限
const db = cloud.database()  // 以管理员身份操作数据库

// 自己实现权限检查
const trip = await db.collection('trips').doc(tripId).get()
const isOwner = trip.data._openid === OPENID
const isCollab = trip.data.collaboratorOpenids?.includes(OPENID)

if (!isOwner && !isCollab) {
  return { error: '没有权限' }
}

// 通过权限检查，才能操作数据
await db.collection('trips').doc(tripId).update({ data: patch })
```

## 什么是 openid

openid 是微信给每个用户分配的唯一标识符。

```
用户A 打开行册 → openid: oXXXX_aaaaa
用户B 打开行册 → openid: oXXXX_bbbbb
```

特点：
- 同一个用户在同一个小程序里，openid 永远不变
- 不同小程序里，同一个用户的 openid 不同
- openid 不是真实姓名，是一串随机字符
- 前端无法伪造（由微信服务器颁发）

行册用 openid 来区分「这条数据是谁的」：

```
trips 集合里的一条数据：
{
  _id: "abc123",
  _openid: "oXXXX_aaaaa",  ← 谁创建的
  name: "南京五日游",
  ...
}
```

## 云函数 vs 前端直接请求数据库

| | 云函数 | 前端直接查询 |
|--|-------|------------|
| 权限 | 管理员权限 | 受权限规则限制 |
| 安全性 | 逻辑在服务器，安全 | 逻辑在客户端，可被绕过 |
| 复杂查询 | 可以做多集合联查 | 受限制 |
| 速度 | 稍慢（多一次网络往返）| 直接 |

行册的策略：
- **只读、简单查询**：部分操作（如 watch 监听）直接从前端访问数据库
- **写操作**：全部通过云函数，保证权限控制

## 云函数的文件结构

```
cloudfunctions/ensure-user/
├── index.js       ← 云函数入口
└── package.json   ← 依赖（这个云函数独立安装自己的依赖）
```

每个云函数有自己独立的 `package.json`，互不干扰。

上传云函数：
1. 右键点击云函数文件夹
2. 「上传并部署：云端安装依赖」

## 小结

- 云开发 = 微信托管的后端（无需自己搭服务器）
- 三大能力：云函数、云数据库、云存储
- openid = 用户唯一标识（由微信颁发）
- 权限控制：通过云函数实现自定义权限逻辑
