# 第四章 · 第三方服务

## 什么是第三方服务

应用开发中，很多功能不需要自己从头做，可以调用专业公司提供的 API（接口）。

行册用到两个第三方服务，都来自高德地图：
- **地点搜索 API**：输入关键词，返回 POI（兴趣点）列表
- **天气预报 API**：输入城市代码，返回天气信息

---

## API 是什么

API（Application Programming Interface，应用程序接口）就是：  
**你发一个 HTTP 请求，对方返回数据。**

就像点外卖：
- 你：发请求（「要一份麻婆豆腐，餐厅是 A 号」）
- 服务商：处理请求（厨师做菜）
- 你：收到响应（外卖送来了）

```
GET https://restapi.amap.com/v5/place/text?keywords=玄武湖&key=YOUR_KEY

响应：
{
  "status": "1",
  "pois": [
    {
      "name": "玄武湖",
      "location": "118.798,32.074",
      "address": "南京市玄武区",
      "adcode": "320102"
    }
  ]
}
```

---

## 高德地图 API

高德（Amap）是阿里旗下的地图服务，在国内使用量最大。

官网：https://lbs.amap.com

使用步骤：
1. 注册高德开发者账号
2. 创建应用，获取 API Key（一串密钥）
3. 在请求里带上这个 Key

```
https://restapi.amap.com/v5/place/text?
  keywords=玄武湖&     ← 搜索关键词
  key=abcdef123456&    ← API Key（标识是谁在用）
  city=南京            ← 城市限制
```

## 行册中的使用方式

### 架构

```
前端（小程序）
    ↓ Taro.cloud.callFunction('amap-poi-search', { keyword })
云函数（服务器）
    ↓ axios.get('https://restapi.amap.com/...')  携带 API Key
高德 API 服务器
    ↓ 返回搜索结果
云函数
    ↓ 处理、过滤数据
前端（更新界面）
```

**为什么要通过云函数中转？**

直接在前端调高德 API 会暴露 API Key：
```javascript
// ❌ 危险：Key 在前端代码里，任何人都能提取出来盗用
const res = await axios.get(`https://restapi.amap.com?key=YOUR_SECRET_KEY&...`)
```

通过云函数中转，Key 只存在服务器端（通过环境变量），用户看不到：
```javascript
// ✅ 安全：前端不知道 Key 是什么
await cloud.callFunction('amap-poi-search', { keyword: '玄武湖' })
```

## 地点搜索（POI Search）

### 云函数代码

```javascript
// cloudfunctions/amap-poi-search/index.js
const axios = require('axios')

exports.main = async (event) => {
  const { keyword, city = '' } = event
  const AMAP_KEY = process.env.AMAP_KEY

  const { data } = await axios.get('https://restapi.amap.com/v5/place/text', {
    params: {
      key: AMAP_KEY,
      keywords: keyword,
      city,
      page_size: 20,
      show_fields: 'business',  // 获取更多信息
    }
  })

  if (data.status !== '1') {
    return { results: [] }
  }

  // 把高德格式转成行册格式
  const results = (data.pois || []).map(poi => {
    const [lng, lat] = (poi.location || ',').split(',').map(Number)
    return {
      name: poi.name,
      address: poi.address || '',
      city: poi.cityname || '',
      adcode: poi.adcode || '',
      lat: lat || 0,
      lng: lng || 0,
    }
  })

  return { results }
}
```

### 前端调用

```typescript
// src/utils/cloud.ts
async searchPoi(params: { keyword: string; city?: string }) {
  return call<..., { results: PoiResult[] }>('amap-poi-search', params)
}

// src/components/SpotSearch/index.tsx
const results = await cloud.searchPoi({ keyword: '玄武湖', city: '南京' })
```

### 经纬度坐标系

高德地图用的是 **GCJ-02**（国测局坐标系，俗称「火星坐标」）。  
这是中国法规要求，和 GPS 坐标（WGS-84）有偏移。

行册存储的经纬度都是 GCJ-02 格式，直接传给高德地图/微信地图使用，不需要转换。

## 天气获取

### 云函数代码

```javascript
// cloudfunctions/amap-weather/index.js
exports.main = async (event) => {
  const { adcode } = event  // 城市代码，如南京 = '320100'
  const AMAP_KEY = process.env.AMAP_KEY

  const { data } = await axios.get('https://restapi.amap.com/v3/weather/weatherInfo', {
    params: {
      key: AMAP_KEY,
      city: adcode,
      extensions: 'all',  // 获取预报（不是当前天气）
    }
  })

  if (data.status !== '1' || !data.forecasts?.length) {
    return { weather: null }
  }

  const forecast = data.forecasts[0]
  const tomorrow = forecast.casts[0]  // 第一条是今天（或明天）

  return {
    weather: {
      city: forecast.city,
      cityAdcode: adcode,
      high: parseInt(tomorrow.daytemp),
      low: parseInt(tomorrow.nighttemp),
      description: tomorrow.dayweather,
      icon: mapWeatherToEmoji(tomorrow.dayweather),
      fetchedAt: new Date().toISOString(),
    }
  }
}
```

### 天气缓存（client-side）

天气数据不需要频繁更新，行册在前端做了缓存：

```typescript
// src/utils/weather.ts
const cache = new Map<string, { data: DayWeather; fetchedAt: number }>()
const TTL = 30 * 60 * 1000  // 30分钟有效期
const MAX_SIZE = 50          // 最多缓存 50 个城市

export async function loadWeather(adcode: string, force = false) {
  const now = Date.now()
  const cached = cache.get(adcode)

  // 有缓存且未过期？直接返回
  if (!force && cached && now - cached.fetchedAt < TTL) {
    return cached.data
  }

  // 需要请求
  const result = await cloud.getWeather({ adcode })

  // 容量满了？删掉最老的
  if (cache.size >= MAX_SIZE) {
    const oldestKey = cache.keys().next().value
    cache.delete(oldestKey)
  }

  cache.set(adcode, { data: result.weather, fetchedAt: now })
  return result.weather
}
```

这是「LRU 缓存」（Least Recently Used，最近最少使用）的简化版本。

## adcode：城市行政区代码

高德地图的 `adcode` 是行政区划代码，规则：
- 省级：`320000`（江苏）
- 市级：`320100`（南京）
- 县区级：`320102`（玄武区）

判断是否为城市级：
```typescript
// src/components/DestinationPicker/index.tsx
// adcode 以 '00' 结尾 = 城市级别
const cities = results.filter(r => r.adcode && r.adcode.endsWith('00'))
```

## 环境变量：保护 API Key

云函数里的 API Key 不应该硬编码在代码里（这样提交 Git 后就泄露了）。正确做法是用环境变量：

```javascript
// 读环境变量
const AMAP_KEY = process.env.AMAP_KEY
```

在微信云开发控制台设置环境变量：
1. 云开发控制台 → 云函数 → 对应函数 → 配置
2. 添加环境变量 `AMAP_KEY = 你的高德Key`

这样代码里没有 Key，Key 只存在服务器配置里。

## 错误处理

调用第三方 API 可能失败（网络问题、Key 过期等），需要优雅处理：

```typescript
// src/views/ItineraryView/DayHeader.tsx
const weather = await loadWeather(adcode).catch(err => {
  console.error('[weather]', err)
  return null  // 失败了就返回 null，界面不显示天气
})
```

```scss
/* 天气显示：有就显示，没有就不占空间 */
{weather && <Text className='dh-weather'>{weather.description} {weather.high}°/{weather.low}°</Text>}
```

## 小结

| 服务 | 用途 | 云函数 | 前端工具 |
|------|------|--------|---------|
| 高德 POI 搜索 | 搜索地点坐标 | `amap-poi-search` | `cloud.searchPoi()` |
| 高德天气 | 显示行程天气 | `amap-weather` | `loadWeather()` |

关键原则：
1. API Key 必须放服务端，不能放前端
2. 对第三方数据要做错误处理（服务可能不可用）
3. 合理缓存，减少 API 调用次数（节省费用）
