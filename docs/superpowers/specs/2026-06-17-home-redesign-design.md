# 首页重设计（四主题）设计文档

状态：已评审，待实现
日期：2026-06-17

## 1. 背景与目标

1.1. 当前首页问题

1.1.1. 顶部「行册」品牌 logo 与各主题专属装饰（手紙期刊号/标语、杂志 VOL./EDITORIAL 报头、护照 PASSPORT/No. 封面、极简 CHRONICLE）占据首屏，信息价值低。

1.1.2. 区块过多且层级不清：装饰头 → AI Banner → 精选攻略横排 → 行程列表 → 已归档折叠区 → 底部 CTA，共 6 段，用户难以一眼分清主次。

1.1.3. 精选攻略（攻略库入口）永远钉在底部、呈现单薄，像广告位。

1.1.4. 已归档折叠区在「单人攻略数很少」的现阶段属于过度设计。

1.2. 本次目标

1.2.1. 删除品牌 logo 与装饰物，让首页更透气。

1.2.2. 重排信息架构，分区清晰、主次分明。

1.2.3. 把「攻略库」从底部广告条重塑为有编辑感、常驻、多条的内容版块，并改名。

1.2.4. 删除归档分区，行程合并为单一列表。

1.2.5. 四主题统一为同一套信息骨架，仅换皮。

## 2. 设计原则

2.1. 四主题共用同一套 DOM 结构与分区顺序，差异只体现在皮肤（色彩、字体、卡片质感）。

2.2. 创建入口优先：首屏重心是「开始一段旅行」。

2.3. 内容版块用「强标题 + 封面内容卡」建立分量，避免广告条观感。

2.4. 不依赖照片、不依赖 emoji（契合全 App 清 emoji 方向，图标走自绘线条 + CSS mask）。

2.5. YAGNI：不为尚未到来的规模提前造收纳/归档机制。

## 3. 信息架构（从上到下）

3.1. 顶部极简栏

3.1.1. 左上角留空（不放问候语、不放品牌 logo）。

3.1.2. 右上角保留头像入口（复用 `AvatarEntry`，通往「我的」）。

3.2. 创建台

3.2.1. 两个并排对等的入口 tile：「AI 规划」与「手动新建」。

3.2.2. 「AI 规划」触发 `AIInterview`（沿用 `onAITrip`）；「手动新建」跳转 new-trip 页（沿用 `onNewTrip`）。

3.2.3. 取代原 `HomeAIBanner`；底部不再放 `HomeBottomCTA`，创建入口全部上移至此。

3.3. 旅人精选（原「攻略库」入口）

3.3.1. 位于创建台之下、我的行程之上，横向滑动，常驻、对所有用户展示、多条。

3.3.2. 卡片采用「生成色封面 + 大字目的地名」：上半部为确定性生成的渐变色块（零照片成本），叠大字号城市名与地区/季节小字；下半部白底显示「目的地名 · 天数晚数 · 地点数」。

3.3.3. 版块标题右侧「更多 ›」进入完整的「旅人精选」页面（原 library 页）。

3.4. 我的行程

3.4.1. 单一统一列表，不再拆分 active/archived。

3.4.2. 进行中（live）行程显示状态标签；即将出发（pre）与已结束（post）不显示标签。

3.4.3. 已结束行程整卡视觉淡化并沉到列表底部。

## 4. 关键交互与逻辑

4.1. 行程排序

4.1.1. 去掉「最近打开/更新（updatedAt）置顶」逻辑（当前杂志主题按 updatedAt 倒序，需移除）。

4.1.2. 分组优先级：live → pre → post。

4.1.3. 组内排序：live 与 pre 按 startDate 升序（最近的在前）；post 按 endDate 降序（最近结束的在前）。

4.2. 状态标签规则

4.2.1. live：渲染 `TripPhaseChip` 的 LiveChip。

4.2.2. pre 与 post：不渲染标签（首页列表只为 live 渲染 `TripPhaseChip`）。

4.2.3. post 的「已结束」语义由整卡淡化 + 沉底 + 日期共同传达。

4.3. 旅人精选取色

4.3.1. 主色由目的地确定性生成：对 `city`（优先）或 `region` 做哈希，映射到一组预设旅行色板（参考现有 postcard `STAMP_COLORS` / tegami `COLORS` 思路），保证同一目的地颜色稳定。

4.3.2. 季节微调：若 `seasons[0]` 存在，对主色做冷暖/饱和度微调（春偏绿、夏偏蓝、秋偏橙、冬偏冷蓝等）；缺失则直接用主色。

4.3.3. 输出渐变二元组 `[c1, c2]`，供封面背景使用；逻辑对季节缺失健壮，不得因无季节而出错。

4.4. 季节标签

4.4.1. `TemplateMeta.seasons: string[]` 字段已存在（注释标注可空），无需修改 schema。

4.4.2. 运营侧在 `trip_templates` 文档中逐步补充 `seasons`（可选、渐进），既丰富配色也利于库页筛选；library 页已有季节筛选维度。

4.4.3. 前端不做季节自动推断；缺失即按 4.3.2 回退。

## 5. 命名变更

5.1. 「攻略库」统一改名为「旅人精选」，涉及：

5.1.1. `HomeFeaturedRow` 版块标题（现「精选攻略」）。

5.1.2. library 页面标题与导航栏标题（`app.config.ts` 中对应 `navigationBarTitleText`）。

5.1.3. 其余出现「攻略库 / 精选攻略」的用户可见文案。

## 6. 组件改动清单

6.1. 新增

6.1.1. 创建台双 tile 组件（如 `HomeCreateTiles`）：共用骨架，主题换皮。

6.1.2. 取色工具（如 `utils/template-cover.ts` 中 `templateCoverColors(card) => [c1, c2]`）：实现 4.3。

6.2. 改造

6.2.1. `pages/home/index.tsx`：移除 active/archived 的 `useMemo` 拆分，改为传全部 trips；新增 4.1 排序；调整传入 props。

6.2.2. `pages/home/shared.tsx`：`HomeViewProps` 移除 `archivedTrips`；`trips` 含全部并已排序。

6.2.3. 四个 `Home{Tegami,Magazine,Postcard,Minimal}.tsx`：重构为新骨架——移除装饰头、`BrandLogo`、`HomeAIBanner`、`HomeBottomCTA`、`HomeArchiveSection` 引用；接入创建台、旅人精选、单一列表；列表项仅 live 渲染标签、post 淡化。

6.2.4. `HomeFeaturedRow`：升级为生成色封面卡（含 4.3 取色），标题改「旅人精选」。

6.2.5. `pages/library/index.tsx` 与 `app.config.ts`：改名文案。

6.3. 删除

6.3.1. `HomeArchiveSection` 组件及其 scss（`home/styles/home-archive.scss`）。

6.3.2. 首页对 `HomeBottomCTA`、`HomeAIBanner`、`BrandLogo` 的使用（组件本身若全仓再无引用则一并删除，否则仅删首页引用）。

## 7. 四主题换皮约定

7.1. 创建台 tile、旅人精选封面卡、行程卡的视觉表达各主题不同（色彩、字体、边框、纹理），但 DOM 结构与分区顺序一致。

7.2. 主题差异通过 CSS 变量与各主题 scss 实现，沿用现有 `theme-class` 机制；不在分发器里写主题条件分支。

## 8. 数据与约束

8.1. 旅人精选模板与示例仅使用中国大陆目的地（高德 POI 当前不支持境外，境外内容会误导且无法落地）。

8.2. 模板封面图（`coverImages`）现阶段为空，本次不依赖真实图片，全部用生成色封面。

8.3. 行程数据源、协作、保存逻辑不变，本次仅改首页展现层。

## 9. 验收标准

9.1. 四主题首页均无「行册」logo 与原装饰头。

9.2. 顶部左上角为空，右上角为头像入口。

9.3. 创建台双 tile 可分别进入 AI 访谈与手动新建；首页无底部 CTA。

9.4. 旅人精选常驻显示、多条、横滑、卡片为生成色封面 + 大字目的地，缺季节也正常显示；「更多」进入旅人精选页。

9.5. 我的行程为单一列表；进行中显示标签，即将出发/已结束无标签；已结束淡化并沉底。

9.6. 行程按 4.1 规则排序，不再出现「最近打开置顶」。

9.7. 全仓「攻略库 / 精选攻略」用户可见文案均改为「旅人精选」。

9.8. 示例与默认内容无境外目的地。

## 10. 不在本次范围（YAGNI）

10.1. 真实封面图上传与展示。

10.2. 归档/收纳功能（待真实用户积累大量过往行程后再议）。

10.3. 最近打开/智能排序。

10.4. 季节自动推断。
